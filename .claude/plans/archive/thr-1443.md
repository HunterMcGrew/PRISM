# Branch Plan: feature/hmcgrew/thr-1443-link-types-and-helpers

## Goal

Introduce `LinkTarget` as a shared type, consolidate link type guards into the type-guards module, and fix a bug where undefined values were being spread as explicit keys on link objects.

---

## Decisions

- `type-guards/link.ts` imports `ILink` from `interfaces/link.ts` (one direction only) — avoids circular dependency.
- `interfaces/link.ts` imports `LinkTarget` from `type-guards/link.ts` (one direction only) — `LinkTarget` is the single source of truth, do not duplicate it inline on `ILink.target`.
- `isLinkObject` lives in `type-guards/link.ts`, not `interfaces/link.ts` — consistent type-guard location, avoids circular dep; typed as `(v: unknown): v is ILink` (broader than `LinkValue`) to match the guard pattern.
- `isLinkObject` checks `typeof href === "string"` in addition to key presence — `{ href: 42 }` is not a valid `ILink`.
- `setLinkTarget` only auto-injects `rel: "noopener noreferrer"` when `target === "_blank"` — not for other target values.
- When `target` is `undefined`, `setLinkTarget` deletes both `target` and `rel` — do not spread `undefined`; the `rel` argument is ignored in this case, callers must use `setLinkRel` instead.
- Same delete-not-spread pattern applies to `setLinkRel` and `setLinkLabel` — absent values must be removed as keys, not set to `undefined`.

## Debugged Issues

### TS2322 — `in` operator on `LinkValue` return type
- **Status:** `fixed`
- **File:** `frontend/lib/interfaces/__tests__/link.test.ts:66,67,74,75,104,124`
- **Root cause:** `setLinkTarget`, `setLinkRel`, `setLinkLabel` return `LinkValue` (`string | ILink`); TypeScript rejects `"key" in result` because `string` is not a valid right-hand operand for `in`.
- **Recommended fix:** Cast `result` to `ILink` at the assertion sites where the input is provably an object — `as ILink`.
- **Suggested tests:** None needed — the existing structural `toEqual` assertions already verify the shape.
- **Fixed in:** `frontend/lib/interfaces/__tests__/link.test.ts` — added `as ILink` on four `result` assignments.
