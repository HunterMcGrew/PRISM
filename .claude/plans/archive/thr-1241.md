# Plan: THR-1241

## Ticket

https://linear.app/tractru/issue/THR-1241/refactor-equipment-specsandcompare-to-utilize-server-components

## Goal

Refactor Equipment SpecsAndCompare to follow the RSC-first pattern established by THR-1240 (Addons): extract types, add explicit client directives, fix accessibility, and clean up the server/client boundary.

---

## Decisions

- `SpecsAndCompare` remains `"use client"` ‚Äî manages dynamic compare state (add/remove/swap items across 1-3 slots), which is inherently interactive. Matches the Addons pattern.
- `CompareCardDropdown` gets explicit `"use client"` ‚Äî uses `useState` and HeadlessUI `Menu`, must not rely on parent's directive for correctness.
- `CompareCard` and `ItemSpecifications` are presentational ‚Äî no hooks, no directives needed. They inherit the client boundary from `SpecsAndCompare` at runtime but are structurally RSC-safe.
- Types (`CompareItem`, `DropdownItem`, `CompareItemState`) extracted to `specs-and-compare/types.ts` ‚Äî server files (`fetchers.ts`, `utils.ts`) currently import from the `"use client"` `SpecsAndCompare.tsx`, which is incorrect. Types should live in a standalone file.
- No Suspense boundaries ‚Äî user-initiated fetches via route handler + `useState` is the established pattern (matches Addons). Suspense is better suited for render-time data loading, not user-triggered compare swaps.
- Fix `<span onClick>` ‚Üí `<button>` for add-compare action ‚Äî keyboard inaccessible as-is.
- Add `aria-label` to remove button on `CompareCard` ‚Äî screen readers need context for icon-only buttons.

### Task 2: Add explicit `"use client"` to `CompareCardDropdown.tsx`

Add the directive to line 1. This component uses `useState` and HeadlessUI `Menu` ‚Äî it must declare its own client boundary per codebase rules.

- `SpecsAndCompare.tsx` lines 202-208: Replace `<span onClick={handleAddCompareCard}>` with `<button type="button" aria-label="Add model to comparison">`. Remove `hover:cursor-pointer` (buttons already have pointer cursor).
- `CompareCard.tsx` line 44: Add `aria-label={`Remove ${name} from comparison`}` to the remove button.

### Task 4: Verify types and tests

- Run `pnpm run test` in frontend for any existing specs-and-compare tests
- Run `tsc --noEmit` to confirm no type regressions
- Run build to confirm no bundle issues
