# Plan: thr-1448

## Ticket

https://linear.app/tractru/issue/THR-1448/mega-menu-editor-admin-shell

## Goal

Build a standalone Gutenberg-based Mega Menu editor as a custom WordPress admin page with draft/publish workflow, revision history, and preview capabilities.

---

## User Stories

- As a content author, I want a dedicated Mega Menu editor in wp-admin so that I can edit mega menu content using the same block editor I use elsewhere.
- As a content author, I want to save a draft before publishing so that I can stage changes without affecting the live site.
- As a content author, I want to preview draft content so that I can verify the mega menu looks correct before publishing.
- As a content author, I want to restore a past revision so that I can recover from an unintended change.
- As a developer, I want mega menu blocks restricted to the mega menu editor so that content authors cannot accidentally insert them in posts or pages.

---

## Decisions

- Editor uses `wp_options` for storage (draft, published, meta, snapshots) rather than custom post types — simpler, no CPT overhead for a single-entity resource.
- Preview uses transient-based tokens (8h TTL) so the frontend iframe can load draft content without WP auth cookies.
- Custom admin shell hides WP sidebar/toolbar via CSS to provide a full-screen editor experience.
- `MEGA_MENU_ALLOWED_BLOCKS` lives in `frontend/lib/constants/mega-menu.ts` — leaf module, no side effects, matches PR #1612 pattern. All consumers import from `@frontend/lib/constants/mega-menu`.
- `getDraftSerializedBlocks()` and `getDefaultSerializedBlocks()` are owned by `MegaMenuBlocksApi` as public static methods — the single source of truth. `MegaMenuEditor` calls them as a consumer. Prevents silent divergence of draft resolution logic and default block markup.
- `DRAFT_OPTION_KEY` and `PUBLISHED_OPTION_KEY` live only in `MegaMenuBlocksApi` — the storage owner. No other class duplicates option key strings.
- Review skill duplication threshold updated: data/logic duplication over shared state flags at 2 sites, pattern duplication stays at 3+.
- `MegaMenuEditor::checkIfMegaMenuIsEnabled()` delegates to `FeatureFlags::is_enabled('enable_mega_menu')` — no direct `get_option` calls for feature flags. PR #1728 (THR-1546) must merge into this branch before #1626 merges to main.

## Acceptance Criteria

### Behavioral

- [ ] Given the "Enable Mega Menu" toggle is off in WordPress admin under Appearance → Platform Settings,
      When a user views the WordPress admin sidebar,
      Then no "Mega Menu" menu item appears.

- [ ] Given the "Enable Mega Menu" toggle is on in Platform Settings,
      When a user navigates to the WordPress admin sidebar,
      Then a "Mega Menu" item appears in the left navigation menu.

- [ ] Given a user clicks the "Mega Menu" admin page,
      When the page loads,
      Then a full-screen Gutenberg-based editor appears with a toolbar at the top, a left sidebar (list view and block inserter), a central canvas area, a right sidebar (block inspector), and a footer breadcrumb.

- [ ] Given a user makes changes in the editor,
      When they click "Save Draft",
      Then the draft saves without publishing, the page can be reloaded, and the same draft content is still present.

- [ ] Given a user has a saved draft,
      When they click "Publish",
      Then the current draft becomes the live mega menu content.

- [ ] Given a user has no saved draft yet,
      When they click "Publish",
      Then an error notification appears explaining that a draft must be saved before publishing.

- [ ] Given a user has a saved draft,
      When they click the preview button (external link icon) in the toolbar,
      Then a new browser tab opens showing a preview of the draft mega menu content.

- [ ] Given the mega menu has been published at least once,
      When a user clicks the revisions button (clock icon) in the toolbar,
      Then a dropdown appears showing up to 3 past published revisions with the publisher name and date.

- [ ] Given the revisions dropdown is open,
      When a user clicks "Restore" on a revision,
      Then that revision's content loads as the current draft in the editor.

- [ ] Given a user clicks the reset button (rotate icon) in the toolbar,
      When the confirmation modal appears,
      Then the Reset button remains disabled until the user types "I UNDERSTAND" exactly; typing anything else keeps it disabled.

- [ ] Given the reset confirmation modal is open and "I UNDERSTAND" has been typed,
      When the user clicks "Reset",
      Then the editor canvas clears to the default empty template.

- [ ] Given the reset confirmation modal is open,
      When the user clicks Cancel or presses Escape,
      Then the modal closes without making any changes to the editor content.

- [ ] Given the revisions dropdown button has keyboard focus,
      When the user presses Enter to open the dropdown,
      Then keyboard focus moves into the dropdown list; pressing Escape closes the dropdown and returns focus to the revisions button.

- [ ] Given a request is made to the public REST endpoint for mega menu blocks,
      When nothing has been published yet,
      Then an empty blocks array is returned (not an error response).

- [ ] Given the mega menu has been published,
      When a request is made to the public REST endpoint for mega menu blocks,
      Then the published block content is returned.

- [ ] Given the mega menu editor is open and the block inserter is opened from the left sidebar,
      When a user browses the available blocks,
      Then only mega menu blocks appear; standard post/page blocks (paragraph, image, etc.) that are not in the allowed list are not shown.

- [ ] Given a content author is editing a standard post or page,
      When they open the block inserter,
      Then the mega-menu block category does not appear.

### Non-behavioral

- [ ] The "Mega Menu" admin menu item only appears when the "Enable Mega Menu" setting is enabled in Platform Settings
- [ ] Admin notices on the Mega Menu editor page (security warnings, update notices) are not hidden or suppressed by editor CSS
- [ ] All interactive elements (Save Draft, Publish, Preview, Revisions, Reset) are keyboard-accessible and operable with Enter or Space
- [ ] The revisions list stores a maximum of 3 past published revisions
- [ ] The preview link opens in a new tab and does not require the viewer to be logged in to WordPress
