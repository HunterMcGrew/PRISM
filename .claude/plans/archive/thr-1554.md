## Debugged Issues

### mega-menu-pane frontend registration removed
- **Status:** fixed
- **Fixed in:** `block-views.tsx` (dynamic import) + `block-registry.ts` (registerBlock with childrenAsRoot: true)
- **Severity:** High
- **Environment:** Frontend (Next.js) — all sites using the mega menu
- **File:** `frontend/lib/services/block/block-registry.ts`
- **Root cause:** Commit `194f32a73` ("chore: remove thrive-mega-menu-renderer and stale block wrappers") accidentally removed the mega-menu-pane registration alongside the intentional renderer removal
- **Steps to Reproduce:**
  1. Navigate to a page with the mega menu on the frontend
  2. Open a navigation item with pane content
  3. Pane children do not render
- **Expected behavior:** Pane content (columns, banners, links) renders inside the mega menu dropdown
- **Actual behavior:** Pane block is skipped by the renderer — no children render
- **Recommended fix:** Re-add dynamic import in `block-views.tsx` and `registerBlock` call in `block-registry.ts`
- **Suggested tests:** Verify pane renders with children, no console errors for unregistered blocks
- **Linear:** `synced`
