# Plan: THR-1467

## Ticket

THR-1467 â€” Homepage button group items not displaying labels on mobile when no icon is configured.

## Goal

When a Button Group is configured with `labelVisibility = "mobile-friendly"`, only hide the item's label on mobile if an icon is also present. Without an icon, a hidden label would leave a visually empty button on small screens.

---

## Decisions

- `labelVisibility === "mobile-friendly"` requires an `icon` before `"hidden sm:block"` is applied to the label span. Removing this guard re-introduces the original bug â€” empty buttons on mobile.
- Fix lives in `frontend/blocks/button-group-item/ButtonGroupItemBlock.tsx:140` (the className ternary inside the label `<span>`).
- Tests mock `useButtonGroupContext` (not the provider) so context variations can be set per case without re-wiring React's context tree. `ButtonGroupItem` and `ButtonGroupItemIcon` are mocked to thin stubs so the test asserts only on the label span.
- Storybook story wraps each case with `ButtonGroupBlock` as a decorator so `ButtonGroupContext` is provided via the real path â€” avoids drift between the block context shape and the story's assumptions.

### TypeScript errors in test file (over-narrow `as const`)
- **Severity:** `major`
- **Status:** `fixed`
- **File:** `frontend/blocks/button-group-item/__tests__/ButtonGroupItemBlock.test.tsx`
- **Problem:** `defaultContext()` used `as const` on literal values, narrowing the return type to specific literals. Overrides passing `"mobile-friendly"`, `"never"`, `"rounded"` failed assignment. Also used `"arrow-right"` (not a valid `CTA_ICONS` key), `"standard"` (not a valid `BUTTON_GROUP_STYLES` key), and `"primary"` (not a valid `BUTTON_GROUP_THEMES` key).
- **Suggested fix:** Replaced the inline type with an explicit `ButtonGroupContextValue` return type composed from `keyof typeof` imports of the schema constants, and switched literals to `"call"`, `"pill"`, `"light"`. Fixed.

## Acceptance Criteria

### Behavioral
- [ ] Given a Button Group with label visibility set to "Mobile-friendly" and no icon configured on an item, When the site is viewed on a mobile viewport (under 640px), Then the button's label text is visible.
- [ ] Given a Button Group with label visibility set to "Mobile-friendly" and an icon configured on an item, When the site is viewed on a mobile viewport, Then the button's label text is hidden and the icon is visible.
- [ ] Given a Button Group with label visibility set to "Mobile-friendly", When the site is viewed on a desktop viewport (640px and above), Then the button's label text is visible regardless of whether an icon is configured.
- [ ] Given a Button Group with label visibility set to "Always", When viewed on any viewport, Then the button's label text is visible.
- [ ] Given a Button Group with label visibility set to "Never", When viewed on any viewport, Then the button's label text is hidden.
- [ ] Given a Button Group with style set to "Rounded", When label visibility is set to "Mobile-friendly", Then the label visibility classes are not applied (rounded style takes precedence).

### Non-behavioral
- [ ] Unit tests cover all four label-visibility modes and the rounded-style bypass.
- [ ] Storybook story exists for `ButtonGroupItemBlock` covering the visibility combinations.
