## Debugged Issues

### Mega Menu Renderer not gated behind feature flag
- **Status:** `fixed`
- **Fixed in:** Added `gravity-platform-core/mega-menu-renderer` to `gated_blocks` in `FeatureFlags.php:19-23`. Updated existing Pest test.
- **Severity:** High
- **Environment:** WordPress admin block editor — all sites
- **File:** `backend/plugins/gravity-platform-core/includes/Settings/FeatureFlags.php:19-23`
- **Root cause:** `gravity-platform-core/mega-menu-renderer` is missing from the `gated_blocks` array for the `enable_mega_menu` flag. The block was introduced in THR-1449 (PR #1628, 2026-04-09) after `FeatureFlags.php` was created in THR-1448 (PR #1626, 2026-04-08) — different PRs, and the renderer's PR didn't update the gate list. The THR-1449 plan has no mention of feature flag gating.
- **Steps to Reproduce:**
  1. Navigate to WordPress Admin → Settings → Platform Settings
  2. Ensure the "Enable Mega Menu" feature flag is unchecked (off)
  3. Open any page or post in the block editor
  4. Search for "mega menu" in the block inserter
  5. "Mega Menu Renderer" appears in results
- **Expected behavior:** No mega-menu blocks visible when flag is off
- **Actual behavior:** "Mega Menu Renderer" appears and is insertable
- **Recommended fix:** Add `'gravity-platform-core/mega-menu-renderer'` to the `gated_blocks` array in `FeatureFlags.php`
- **Suggested tests:** Pest test asserting `FeatureFlags::get_gated_blocks('enable_mega_menu')` includes `gravity-platform-core/mega-menu-renderer`
- **Linear:** `synced` (root cause and fix already match ticket — no update needed)
