## Goal

Restore keyboard-only focus ring behavior on mega menu navigation items — mouse clicks should not trigger a visible focus ring.

---

## User Stories

Not applicable — bug ticket.

## Design

Not applicable.

## Decisions

- `outline-none` must be unconditional (not behind `focus-visible:`) because it suppresses the browser's default outline on all focus types. The custom ring (`focus-visible:ring-2`) provides the keyboard-only indicator — the browser default must not leak through on mouse click.
- Both `MegaMenuNavigationItem.tsx` and `NavigationMenuItem.tsx` need the same fix — they share the same focus class pattern for the same visual element.

## Debugged Issues

### Browser default outline leaks through on mouse click due to `focus-visible:outline-none`
- **Status:** `fixed`
- **Fixed in:** `MegaMenuNavigationItem.tsx:26` and `NavigationMenuItem.tsx:61` — changed `focus-visible:outline-none` to `outline-none`
- **Severity:** Low
- **Environment:** Frontend — mega menu navigation items on the live site (all browsers)
- **File:** `frontend/components/navigation-menu-item/MegaMenuNavigationItem.tsx:26` and `frontend/components/navigation-menu-item/NavigationMenuItem.tsx:61`
- **Root cause:** `outline-none` is gated behind `focus-visible:` so it only suppresses the browser default outline on keyboard focus. On mouse click, `:focus` matches but `:focus-visible` does not — the browser's default focus outline renders. The `focus:` → `focus-visible:` migration in THR-1447/THR-1451 treated `outline-none` and `ring-2` as a single unit, but they serve different purposes: `outline-none` suppresses the browser default (should always apply), while `ring-2` adds a custom indicator (should be keyboard-only).
- **Steps to Reproduce:**
  1. Open a page with the mega menu
  2. Click on a top-level navigation menu item with the mouse
  3. Observe browser default focus outline appears around the item
- **Expected behavior:** No visible focus indicator on mouse click
- **Actual behavior:** Browser default focus outline appears on mouse click
- **Recommended fix:** Change `focus-visible:outline-none` to `outline-none` in `NAV_ITEM_BASE_CLASSES` (MegaMenuNavigationItem.tsx:26) and `LINK_CLASSNAMES` (NavigationMenuItem.tsx:61). Keep `focus-visible:ring-2` as-is. The resulting class string should be `outline-none focus-visible:ring-2` — always suppress the browser default, only show the custom ring on keyboard.
- **Suggested tests:** Update existing NavigationMenuItem and MegaMenuNavigationItem tests to assert `outline-none` class is present (not behind `focus-visible:`). Storybook stories should visually verify no focus ring on click.
- **Linear:** `synced`
