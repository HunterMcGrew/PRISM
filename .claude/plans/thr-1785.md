## Review Issues

### Typo: doubled `<source>` path segment in data-layer.md

- **Severity:** `minor`
- **Status:** `fixed`
- **File:** `.claude/architect/data-layer.md:108`
- **Problem:** `<entity>/<source>/<source>/` has `<source>` doubled. Should be `<entity>/<source>/`.
- **Suggested fix:** Change `<entity>/<source>/<source>/` → `<entity>/<source>/` in the services layer rules bullet.

### Stale "protocol" path term in ADR-0021 Decision section

- **Severity:** `minor`
- **Status:** `fixed`
- **File:** `.claude/spec/adrs/0021-data-layer-reshape.md:28`
- **Problem:** The Mapping paragraph says `data-access/<entity>/<protocol>/mappers/`. Every other structural path reference in the PR uses `<source>`, not `<protocol>`. Terminology drifted after the "protocol → source" naming change was made globally elsewhere.
- **Suggested fix:** Change `<protocol>` → `<source>` in that one sentence.

### Stale `repository.ts` filename in boundary rule review table

- **Severity:** `minor`
- **Status:** `fixed`
- **File:** `.claude/rules/data-layer-boundaries.md:76`
- **Problem:** Remediation for "new source added without entity-first folder" says to create `repository.ts`. But the target shape uses named class files (`GQLPromotionRepository.ts`), not a generic `repository.ts`. Leftover from when repos were factory functions returning object literals.
- **Suggested fix:** Change `repository.ts` → `<Source><Entity>Repository.ts` (e.g. `GQLPromotionRepository.ts`) in the remediation text, or simplify to "per `data-layer.md`" without listing filenames.

### `import "server-only"` absent from Next 16 codemod example

- **Severity:** `minor`
- **Status:** `fixed`
- **File:** `.claude/architect/data-layer.md` (§ Caching, "Next.js 16 target (codemod)" code block)
- **Problem:** The Next 16 `"use cache"` example omits `import "server-only"`, while the "Today (Next.js 15)" example immediately above includes it. The migration prose says "Strip the `unstable_cache` wrapper, add the directive" — no mention of whether `server-only` is kept. A dev following the codemod instructions might omit it.
- **Suggested fix:** Either add `import "server-only"` to the Next 16 example, or add a prose note: "Keep `import 'server-only'` — only strip the `unstable_cache` wrapper and its config args."

### Broken inline code formatting in `gql` template-literal mention

- **Severity:** `minor`
- **Status:** `fixed`
- **File:** `.claude/rules/data-layer-boundaries.md:103`
- **Problem:** The Future enforcement bullet uses `` `gql\`\`` `` — escaped-backtick noise inside a code span — and has no spaces between adjacent inline code spans, so the words run together when rendered.
- **Suggested fix:** Replace with `` `no-restricted-syntax` to ban `gql` template literals outside `data-access/<entity>/<source>/queries/` `` (or the explicit `` `` gql`...` `` `` form with proper double-backtick fencing).
- **Fixed in:** Clove (2026-04-28). Applied the suggested fix verbatim.

### `domain/` framing vs. Story 2's pure rename

- **Severity:** `minor`
- **Status:** `fixed`
- **File:** `.claude/architect/data-layer.md:39-41` (Folder Structure annotation) and `.claude/plans/epic-thr-1784.md` (Story 2 task 1)
- **Problem:** `data-layer.md` annotates `domain/` as "domain models, not contracts," but Story 2 describes `lib/interfaces/` → `lib/domain/` as a "pure rename." A pure rename moves contract types like `BlockResolver` (currently in `lib/interfaces/block.ts`) into `lib/domain/`, contradicting the framing. The resolver example at `data-layer.md:441-442` highlights the tension by importing from `@frontend/lib/factory/ServiceFactory` (target) and `@frontend/lib/interfaces/block` (legacy) in the same snippet.
- **Suggested fix:** Resolve when Story 2 picks up — either accept that `domain/` carries both models and contracts (drop the "not contracts" framing), split contracts into a separate folder (and update Story 2 to reflect the non-pure rename), or update the annotation to clarify scope.
- **Fixed in:** Winston's split-rename call (2026-04-28). `interfaces/` splits into `domain/` (business nouns) + `contracts/` (application contracts), with a sanity-check decision rule documented in `data-layer.md` § Folder Structure. Resolver example import updated. Epic plan Story 2 task 1 reframed as a split-rename. Linear THR-1786 carries the file-by-file split.

### ServiceFactory shape change not flagged

- **Severity:** `minor`
- **Status:** `fixed`
- **File:** `.claude/architect/data-layer.md:421-433` (ServiceFactory example)
- **Problem:** The example uses an object literal (`export const ServiceFactory = { ... }`), but today's `ServiceFactory` is a static class per `frontend-services.md`. The "What Stays the Same" section says "ServiceFactory survives," which is true at the consumer-facing API level, but the shape change is implicit.
- **Suggested fix:** If the object-literal shape is intentional, add a one-line note in the example or in `## What Stays the Same`. If it's incidental, match the existing static-class shape so the architect doc stays directly transcribable.
- **Re-flagged:** 2026-04-28 by Eric on PR #1937 (inline comment on line 429).
- **Fixed in:** Clove (2026-04-28). Matched the existing static-class shape — `export class ServiceFactory { static createPromotionService(site) { ... } }`. The surrounding doc says "Repos and services are both classes," so an object-literal factory would be the odd one out. Now directly transcribable from `frontend/lib/services/ServiceFactory.ts`.

### ServiceFactory file-move task missing from any story

- **Severity:** `minor`
- **Status:** `fixed`
- **File:** `.claude/architect/data-layer.md:86` (Folder Structure tree) and `.claude/architect/manifest.json:19`
- **Problem:** The architect doc puts `ServiceFactory.ts` at `frontend/lib/factory/`, and the manifest registers `frontend/lib/factory/**` for `data-layer.md`. But today the file lives at `frontend/lib/services/ServiceFactory.ts`, and no task in any of the five stories (THR-1785/1786/1788/1789/1790) moves it. Every other location move is explicitly tracked (interfaces split in THR-1786, `rsc/fetchers.ts` in THR-1786 task 3, data-access restructure in THR-1786 task 2). The factory move is the odd one out.
- **Suggested fix:** Add a task to Story 2 (mechanical file move + import path update across consumers) OR make explicit in the doc that `factory/` references are forward-looking and the file currently lives at `services/`.
- **Flagged:** 2026-04-28 by Eric on PR #1937 (inline comment on line 86).
- **Fixed in:** Winston (2026-04-28). Picked option 1 — added a fourth move task to Story 2 in `epic-thr-1784.md`. Reasoning: the factory is the wiring layer, not a service; putting it in `services/` is the same flavor of misleading taxonomy that the `interfaces/` split corrected (location was telling a wrong story about what the file is). The "forward-looking" framing was rejected because it would import interim language INTO the architect doc, which is meant to describe the target shape. Architect doc and manifest stay correctly aspirational; Story 2 closes the gap between aspiration and code. THR-1786 in Linear should be re-synced to reflect the fifth task when picked up.

## Acceptance Criteria

### Behavioral

- [ ] Given an agent loads context for a file under `frontend/lib/data-access/`, `frontend/lib/services/`, `frontend/lib/domain/`, `frontend/lib/factory/`, or `frontend/lib/routing/`, When the manifest matches, Then `data-layer.md` is loaded.
- [ ] Given a developer reading `data-layer.md`, When they reach the end, Then they can describe the target folder structure, name a fetcher in the right shape, and explain why caching lives at the data-access layer.
- [ ] Given a PR introduces a direct `@apollo/client` import outside `frontend/lib/data-access/`, When Eric or Briar reviews, Then the boundary violation is flagged with concrete remediation language.
- [ ] Given a PR adds a new entity, source, or substantial DAL operation, When Eric or Briar reviews, Then the new work is checked against the target shape and the scope rule.

### Non-behavioral

- [ ] ADR-0021 in `accepted` status with all four required sections.
- [ ] `data-layer.md` registered in `manifest.json` for all five relevant directories.
- [ ] `frontend-services.md` and `frontend-blocks.md` reference `data-layer.md` and clearly mark the current shape as interim.
- [ ] No code changes ship from this PR. Migration work tracked under sibling stories.
