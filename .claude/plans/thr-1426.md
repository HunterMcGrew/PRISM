# Branch Plan: robertsamuel/thr-1426-build-a-new-latest-posts-block

## Goal

Replace the WordPress core `core/latest-posts` block with a custom `gravity-platform-core/latest-posts` block, supporting configurable layouts, featured image sizing, content display toggles, and backward compatibility with existing saved blocks.

---

## Decisions

- Custom block registered as `gravity-platform-core/latest-posts`; core block excluded from inserter via `blocks.php`
- `core/latest-posts` alias kept in `block-registry.ts` for backward compatibility with existing content
- `core-latest-posts-adapter.ts` remaps core attribute keys to custom schema before `getBlockProps` merge
- `linkFeaturedImage` feature removed entirely (not in `BlockAttributes`, editor, or component)
- `COLUMN_STYLES` includes `1: "grid-cols-1"` entry with fallback for safety
- `SIZE_TO_ASPECT_RATIO` moved to module scope to avoid re-creation on render
- DOMPurify used for sanitizing `full_post` content in `dangerouslySetInnerHTML`
- Cache key in `get-block-props.ts` uses deterministic filtering parameters only (not `Object.values(resolvedAttrs)`)
- Editor hooks extracted to `use-filter-options.ts` and `use-attribute-handlers.ts`

## Review Issues

### CoreLatestPostsBlockConfig spread order defeats category adapter
- **Status:** open
- **Severity:** Major
- **File:** `frontend/blocks/posts/config.ts:47-52`
- **Problem:** `...ctx.attrs` after `...adaptCoreLatestPostsAttributes(ctx.attrs)` overwrites the adapter's `categories` conversion back to raw `number[]` format
- **Recommended fix:** Swap spread order so adapted values take priority: `{ ...ctx.attrs, ...adaptCoreLatestPostsAttributes(ctx.attrs ?? {}) }`
