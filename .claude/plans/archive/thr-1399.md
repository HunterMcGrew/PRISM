# Branch Plan: feature/hmcgrew/thr-1399-make-urlinput-component-wrapper-2

## Goal

Add `ControlledUrlInput`, move `LinkControl` out of `editor-sidebar`, refactor `ToolbarLink`, and update all block `edit.tsx` consumers to use the new components.

---

## Decisions

- `ControlledUrlInput` is the new canonical wrapper — blocks should use it instead of inline URL input logic.
- `LinkControl` moved from `components/editor-sidebar/` to `components/` — makes it accessible without the editor-sidebar path dependency.
- `ToolbarLink` refactored to use the updated `LinkControl` location and shared `LinkValue` types from `thr-1443`.
- Depends on `feature/hmcgrew/thr-1443-link-types-and-helpers` for `ILink`, `LinkValue`, `LinkTarget`, `isLinkObject`, and the `setLink*` helpers.
- `ControlledUrlInput` keeps `setShowUrlSuggestions(false)` in `onBlur` intentionally — required for correct blur behavior. The touch-click gap (blur fires before tap) is a known trade-off; `mousedown` `event.preventDefault()` on suggestion buttons covers the pointer case.

### `ControlledUrlInput` renders empty `Popover` when all suggestions are filtered out
- **Status:** `fixed`
- **Severity:** Minor
- **File:** `backend/plugins/gravity-platform-core/src/components/editor-sidebar/ControlledUrlInput.tsx:552`
- **Problem:** `__experimentalRenderSuggestions` only guards on `!showUrlSuggestions`. If `suggestions` is non-empty but all fail `isAllowedSuggestion` (e.g., all taxonomy-term results), `filteredSuggestions` is empty, `isLoading` is false, and a visible-but-empty `Popover` is rendered.
- **Recommended fix:** Add early return: `if (!isLoading && filteredSuggestions.length === 0) return null;` after building `filteredSuggestions`.

### `ControlledUrlInput` touch-click swallowed by `onBlur`
- **Status:** `deferred`
- **Severity:** Minor
- **File:** `backend/plugins/gravity-platform-core/src/components/editor-sidebar/ControlledUrlInput.tsx:183`
- **Root cause:** `setShowUrlSuggestions(false)` on `onBlur` unmounts the `Popover` before touch `click` fires; `mousedown` prevention only covers pointer devices.
- **Why deferred:** `onBlur` is intentionally kept — it was required to make the URL input blur correctly. The `mousedown` `event.preventDefault()` on suggestion buttons covers the pointer case. Touch-device gap is a known trade-off.
