# Branch Plan: jmotes/thr-1451-refactor-blocks-renderer-for-better-lazy-loading

## Goal

Refactor the blocks renderer from a monolithic BlockConfig pattern to a resolver + view pattern with separate lazy-loaded modules for better code splitting.

---

## Decisions

- BlockConfig replaced with BlockRegistration (resolver + view lazy loaders) — enables independent code splitting of data resolution and rendering
- Schema files (schema.ts) replace config.ts for BlockAttributes and DEFAULT_ATTRIBUTES exports
- BlockService manages separate resolverCache and viewCache with parallel loading via Promise.all
- `childrenAsRoot` replaces `treatInnerBlocksAsRoot` — cleaner naming
- `import type` used consistently for BlockAttributes in backend edit.tsx files — prevents wp-scripts bundle failures
