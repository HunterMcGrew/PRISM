## Debugged Issues

### ArrowButton renders behind background overlay
- **Status:** `fixed`
- **Severity:** Medium
- **Environment:** Frontend — all viewports, any Panel Hero with CTA link + background opacity
- **File:** `MainPanelBlock.tsx:150`, `SecondaryInnerPanelBlock.tsx:120`, `ArrowButton.tsx:18`
- **Fixed in:** `ArrowButton.tsx:18` — added `relative` to outer div className
- **Root cause:** (Sasha verified) ArrowButton is a non-positioned in-flow element, painted in CSS step 3 (non-positioned descendants). The background overlay is `absolute` with `opacity < 1`, creating a stacking context painted in step 6 (positioned descendants). Step 6 paints after step 3, so the overlay covers the arrow. The sibling text content div has `relative` (also step 6, wins by source order) but ArrowButton does not.
- **Steps to Reproduce:**
  1. Navigate to a page with a Panel Hero block that has a CTA link and background overlay with opacity
  2. Observe the arrow icon appears behind the overlay
- **Expected behavior:** Arrow icon renders in front of the background overlay
- **Actual behavior:** Arrow icon is obscured by the overlay
- **Recommended fix:** Add `relative` to ArrowButton's own outer div (`ArrowButton.tsx:18`) — fixes all usages with a single change. Alternatively, wrap `<ArrowButton />` in a `relative` div in both parent files.
- **Suggested tests:** Visual verification — no unit test needed for a CSS stacking fix
- **Linear:** `synced`
