# Plan: THR-1447

## Ticket

Mega Menu Editor — Root Block + Navigation Item Block

## Goal

Add the MegaMenuEditor root block and MegaMenuNavigationItem child block for building mega navigation menus in the WordPress block editor and rendering them on the Next.js frontend.

---

## Decisions

- `ControlledUrlInput` lives in PR #1616 (THR-1399) — this branch depends on it being merged first.
- MegaMenuEditor is a server component; MegaMenuNavigationItem and MegaMenuContext are client components — needed for Popover/portal interactivity.
- Navigation item panels use `createPortal` to render into a panel zone below the nav bar in the editor, with CSS `order` for positioning.
- `resolver.props.ts` split exists because the backend editor imports it directly for live preview.
- `NAVIGATION_MENU_CLASSES` canonical location is `frontend/blocks/mega-menu-navigation-item/schema.ts` — `MegaMenuEditor.tsx` imports from there.
- `IconSelector` component is added but not currently used by the navigation item edit — it's scaffolding for a future CTA icon feature.
- `MegaMenuEditorProps` decoupled from `BlockAttributes` — component defines only `id`, `children`, `containerStyles`. Resolver bridges the CMS schema gap.
- `MegaMenuProvider` callback props should use `on` prefix (`onCloseMenu`/`onCloseDropdown`) per code-standards handler naming — apply in this branch before merge.

## Acceptance Criteria

### Behavioral

- Given a WordPress editor page with a Mega Menu Editor block, when the editor loads, then 4 default navigation items appear in the menu bar.
- Given a navigation item with type "Link", when a URL is entered in the inspector panel, then the button renders as a link that navigates to the URL on the frontend.
- Given a navigation item with type "Dropdown", when the button is clicked on the frontend, then a dropdown panel opens below the navigation bar.
- Given the dropdown panel is open, when the user presses Escape or clicks outside, then the panel closes.
- Given the mega menu editor block, when background color, title color, hover background color, or title hover color are changed in the inspector panel, then the colors update in the live preview.
- Given a navigation item button, when focused via keyboard, then a visible focus ring appears.
- Given a navigation item button, when focused via mouse click, then no focus ring appears.
- Given the mega menu editor block, when rendered on the frontend, then CSS custom property variables are applied to the root element and inherited by navigation items.
