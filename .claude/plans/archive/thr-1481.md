# Plan: THR-1481

## Ticket

https://linear.app/tractru/issue/THR-1481/dealer-announcements-carousel-duplicate-arrows-appear-after-adding

## Goal

Fix duplicate navigation arrows appearing in the Dealer Announcements Carousel when adding slides via the editor block toolbar, by fully decoupling arrow rendering and interaction from Splide's internal DOM management.

---

## Decisions

- Splide's built-in pagination and arrows are both disabled in `splideOptions` — custom React elements handle rendering and interaction to prevent DOM duplication on Splide re-initialization.
- Custom arrow buttons use explicit `onClick` handlers (`splideRef.current.go('<')`/`go('>')`) instead of relying on Splide's internal class-based binding — this eliminates the coupling that caused the duplication bug and makes the pattern symmetric with how custom pagination already works (`handlePaginationClick`).
- Arrow buttons get `aria-label` attributes for accessibility parity with pagination buttons (which already have them).
