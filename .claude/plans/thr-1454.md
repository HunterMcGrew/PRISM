## Review Issues

### `fetchAllForms` cache / tags (open)

- **Severity:** Minor
- **File:** `frontend/lib/forms/fetch-form.ts`
- **Note:** `fetchForm` sets `next.revalidate` + tags; `fetchAllForms` does not. If `fetchAllForms` is used for invalidation-sensitive flows, consider aligning cache policy.

### `serializeForm` typing (open)

- **Severity:** Minor
- **File:** `frontend/lib/forms/serialize-form.ts`
- **Note:** `as FormFieldValuesInput[]` after stripping files is pragmatic; acceptable if tests cover shape. Optional follow-up: narrow with a type guard or zod.

### `GfForm` pruneState comment (fixed)

- **Severity:** Minor
- **File:** `frontend/components/gravity-forms/GfForm.tsx`
- **Note:** Block comment preserved on `pruneState` in current code — explains the empty-string filter and temporary-fix intent. No further action needed.

### `fieldValues` JSON cast without shape validation (deferred)

- **Severity:** Minor
- **Status:** fixed
- **File:** `frontend/lib/forms/submit-form-action.ts:64`
- **Problem:** `JSON.parse(fieldValuesRaw) as FormFieldValuesInput[]` casts without verifying the parsed value is an array. GraphQL will reject a malformed payload, so risk is low, but an `Array.isArray()` guard would make the error message more actionable.
- **Suggested fix:** Add `if (!Array.isArray(fieldValues)) return { errors: [{ message: "Invalid field values" }] }` after parse. Deferred — out of scope for this ticket.

### `serializeForm` only handles first file per field (open)

- **Severity:** Minor
- **File:** `frontend/lib/forms/serialize-form.ts:21`
- **Note:** `withUpload.fileUploadValues[0]` grabs only the first `File`. Multi-file upload fields would silently drop files 2+. Acceptable if Gravity Forms single-file fields are the only in-scope use case, but worth a comment.

### `IconSelectionGrid` icon buttons missing accessible names (open)

- **Severity:** Minor
- **File:** `backend/plugins/gravity-platform-core/src/components/editor-sidebar/IconSelectionGrid.tsx:71`
- **Note:** Inner icon `<Button>` elements contain only a rendered `<Icon />` with no `aria-label`. Each button needs `aria-label={key}` (or visually hidden text) so screen reader and keyboard users know which icon is which.
