# Plan: THR-1525

## Ticket

https://linear.app/tractru/issue/THR-1525/set-up-internal-docs-site-with-nextra-on-vercel

## Goal

Stand up a password-protected internal docs site built with Nextra, deployed to Vercel with Deployment Protection.

---

## Decisions

- Nextra v4 (App Router) chosen over Docusaurus — team already knows Next.js/React, first-class Vercel support
- Docs site lives in `docs/` at the repo root, not a separate `docs-site/` directory — keeps content colocated
- Content files stay as `.md` (not `.mdx`) — existing docs use angle-bracket URLs and HTML patterns that break the MDX compiler
- Custom `img` component in `mdx-components.js` fixes `Element is not defined` SSR error for markdown images
- `docs` added to `pnpm-workspace.yaml` as a workspace member
- Auth via Vercel Pro Deployment Protection (shared password) — zero code, toggle in dashboard
- Single password for both dev and user docs — support team rewrites user docs for dealers anyway
- Pagefind CLI generates the search index as a post-build step — Nextra 4 expects `/_pagefind/pagefind.js` at runtime but doesn't bundle or run Pagefind itself

- **Status:** `fixed`
- **Severity:** High
- **Environment:** `pnpm build` in docs/
- **File:** `docs/mdx-components.js:3`
- **Root cause:** Nextra v4 renders `.md` image syntax as DOM `Element` during SSR, which doesn't exist in Node
- **Steps to Reproduce:**

- **Expected behavior:** Build succeeds
- **Actual behavior:** `ReferenceError: Element is not defined` during static page generation
- **Recommended fix:** Override `img` in `mdx-components.js` with a plain JSX `<img>` element

### Pagefind search index missing in production
- **Status:** `fixed`
- **Severity:** Medium
- **Environment:** Production — https://docs.thrivewebsiteplatform.com/
- **File:** `docs/package.json:7` (build script)
- **Fixed in:** Added `pagefind` dev dependency + chained `pagefind --site .next/server/app --output-path public/_pagefind` after `next build` in `docs/package.json`. Added `public/_pagefind/` to `docs/.gitignore`.
- **Root cause:** Nextra 4's `<Search>` component dynamically imports `/_pagefind/pagefind.js` at runtime ([search.js:13-15](docs/node_modules/nextra/dist/client/components/search.js)). Pagefind is a build-time CLI tool that crawls the static HTML output of `next build` and generates the `_pagefind/` directory with the search index. The docs site build script is just `next build` — there is no post-build step running `pagefind`, and `pagefind` is not installed as a dependency. The `_pagefind/` directory was never generated, so the dynamic import fails with a `TypeError: Failed to fetch dynamically imported module`.
- **Steps to Reproduce:**
  1. Navigate to https://docs.thrivewebsiteplatform.com/
  2. Click the search box or press Cmd+K / Ctrl+K
  3. Type any search term
- **Expected behavior:** Search returns results from the docs content
- **Actual behavior:** Error message: "Failed to load search index. TypeError: Failed to fetch dynamically imported module: https://docs.thrivewebsiteplatform.com/_pagefind/pagefind.js"
- **Recommended fix:** Install `pagefind` as a dev dependency and add a postbuild step. Update `docs/package.json` scripts to: `"build": "next build && pagefind --site .next/server/app --output-path public/_pagefind"` (the exact `--site` path may vary — Nextra 4 outputs static HTML during `next build` and the Pagefind CLI needs to index those files). Alternatively, for Vercel deployment, add a `postbuild` or `build` override in Vercel project settings. Nextra's own docs recommend running `pagefind --site out` for static exports, but since this site uses the default Next.js server output (not `output: 'export'`), the path to the built HTML pages needs to be verified.
- **Suggested tests:** After fix, run `pnpm build` in `docs/` and verify `public/_pagefind/pagefind.js` exists. Then test search in local preview (`pnpm start`).
