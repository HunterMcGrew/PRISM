# Plan: THR-1427

## Ticket

THR-1427: Redirects upserts aren't triggering cache revalidation

## Goal

Ensure that upserts to the `redirects` post type trigger cache revalidation on the frontend.

---

## Decisions

- `redirects` post type has no ACF fields, so it must be handled by `save_post` (not `acf/save_post`)
- Adding `'redirects'` to `TYPES_WITHOUT_CUSTOM_FIELDS` is the correct and minimal fix
- `singularType = rtrim('redirects', 's')` yields `'redirect'`, so `redirect-{id}` cache tags are correctly formed
- `handleTrashedPost` correctly skips `redirects` — trashing is handled by `handleSavePost` when status changes to `trash`
- `handleDeletePost` already handled all types including `redirects` — no source change needed, but tests were added for coverage
