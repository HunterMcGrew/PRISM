# Plan: THR-1588

## Ticket

https://linear.app/tractru/issue/THR-1588/mega-menu-info-banner-image-is-squished

## Goal

Fix the info banner image in the mega menu so it renders at its correct aspect ratio without distortion.

---

## Decisions

- Scope limited to info banner image rendering only — no other mega menu blocks or layout changes
- Root cause confirmed: missing `object-fit` CSS on the info banner image, combined with missing `mediaDetails` on migrated images causing a 400×400 fallback aspect ratio
- Fix is `object-contain` on the `WithSkeletonImage` className — matches the defensive pattern used by all other mega menu image blocks (`object-cover`), but uses `contain` because this is a logo that must be fully visible
- Migration `normalizeImageData()` updated in this ticket — extracts `title` and `mediaDetails` from legacy data (both IImage and NavigationMenuItemImage shapes). No database calls needed — the legacy data already carries `mediaWidth`/`mediaHeight` and `mediaDetails`
- Info banner link attributes migrated from flat strings (`buttonUrl` + `buttonUrlTarget`) to `link: ILink` — matches the pattern used by all other mega menu blocks (featured-banner, link-with-icon, navigation-list). Mega menu is not in production, so no backward compatibility needed. Migration service outputs `link` as ILink object.

## Debugged Issues

### Block registration not updated for ILink migration — tsc build failure
- **Status:** `fixed`
- **Fixed in:** `index.tsx` — replaced `buttonUrl`/`buttonUrlTarget` registrations with `link: { type: "object" }`
- **Severity:** Critical
- **Environment:** Build — `tsc` in `backend/plugins/gravity-platform-core`
- **File:** `backend/plugins/gravity-platform-core/src/blocks/mega-menu-info-banner/index.tsx:28-35`
- **Root cause:** The ILink migration (commit `72b65f933`) updated schema, resolver, component, editor, and tests but missed the block registration file `index.tsx`. It still registers `buttonUrl` and `buttonUrlTarget` attributes referencing `DEFAULT_ATTRIBUTES.buttonUrl` and `DEFAULT_ATTRIBUTES.buttonUrlTarget`, which no longer exist after `schema.ts` was migrated to `link: ILink`.
- **Steps to Reproduce:**
  1. Run `pnpm run build --filter=gravity-platform-core` (or `tsc` in that package)
  2. Observe three TS errors: TS2769 (no overload match), TS2339 x2 (`buttonUrl`/`buttonUrlTarget` not on `BlockAttributes`)
- **Expected behavior:** TypeScript build passes cleanly
- **Actual behavior:** Three type errors because `index.tsx` references removed attributes
- **Recommended fix:** Replace the `buttonUrl` and `buttonUrlTarget` attribute registrations (lines 28-35) with a single `link: { type: "object", default: DEFAULT_ATTRIBUTES.link }` — matching the `mega-menu-featured-banner/index.tsx` pattern
- **Suggested tests:** None needed — tsc passing is the test
- **Linear:** `not synced`

### Info banner image distorted — missing object-fit
- **Status:** `fixed`
- **Fixed in:** `MegaMenuInfoBanner.tsx` (added `object-contain`) + `MegaMenuMigrationService.php` (added `title` and `mediaDetails` extraction to `normalizeImageData`)
- **Severity:** Medium
- **Environment:** Frontend — mega menu info banner block on live site
- **File:** `frontend/components/info-banner/MegaMenuInfoBanner.tsx:46`
- **Root cause:** The `WithSkeletonImage` className has no `object-fit` property. The default `object-fit: fill` stretches the image content to fill the box, distorting it when the box aspect ratio (determined by HTML width/height attributes) doesn't match the actual image ratio. For migrated images, `normalizeImageData()` in `MegaMenuMigrationService.php:907` omits `mediaDetails`, so the component falls back to 400×400 (1:1 ratio) regardless of the actual image dimensions.
- **Steps to Reproduce:**
  1. Navigate to a dealer site with a mega menu that includes a migrated info banner block with a non-square image
  2. Open the mega menu
  3. Observe the info banner image — it renders with a distorted aspect ratio
- **Expected behavior:** The image renders at its correct aspect ratio without distortion
- **Actual behavior:** The image is squished/compressed because the 1:1 box from the 400×400 fallback doesn't match the actual image ratio
- **Recommended fix:** Add `object-contain` to the `WithSkeletonImage` className in `MegaMenuInfoBanner.tsx:46`. This matches the defensive pattern used by all other mega menu image blocks (MegaMenuLocations uses `object-cover object-center`, MegaMenuFeaturedBanner uses `object-cover`). Use `contain` instead of `cover` because this is a logo/shield that must be fully visible.
- **Suggested tests:** Update the Storybook `LandscapeImage` story to visually verify aspect ratio preservation. Add a test asserting the img has the `object-contain` class.
- **Linear:** `synced`
