## Goal

Remove the non-functional "Stack on Mobile" toggle from the mega-menu-columns editor sidebar and drop the `isStackedOnMobile` attribute from the block's registration, since the mega menu will not be shown on mobile in the current UX.

## Implementation Tasks

### Clove (implementation)

1. Remove the `ToggleControl` for "Stack on Mobile" from `backend/plugins/gravity-platform-core/src/blocks/mega-menu-columns/edit.tsx`, along with its now-unused `ToggleControl` import.
2. Remove the `isStackedOnMobile` attribute declaration from `backend/plugins/gravity-platform-core/src/blocks/mega-menu-columns/index.tsx`.
3. Remove the `isStackedOnMobile: true` seed from the `TEMPLATE` in `backend/plugins/gravity-platform-core/src/blocks/mega-menu-pane/edit.tsx` â€” otherwise the parent would seed an attribute the child no longer declares.

---

## Decisions

- **Superseded (2026-04-21):** The prior "keep the toggle for future-proofing" direction was reversed by product. A visible control that does nothing is worse than no control; removing it today is cleaner than carrying a non-functional toggle. When mobile mega menu UX ships later, the toggle and attribute can be re-added alongside the rendering work.
- Shared `ColumnsBlock.tsx` and `ColumnsBlockAttributes` remain unchanged â€” `core/columns` still uses `isStackedOnMobile` legitimately, and removing the attribute from mega-menu-columns registration does not affect frontend rendering (the component defaults `isStackedOnMobile = true`, so saved content renders unchanged).
- The `isStackedOnMobile` seed in `mega-menu-pane/edit.tsx` TEMPLATE had to go too â€” otherwise WP would seed an attribute that mega-menu-columns no longer declares, producing stale stored attributes that would never be read.
- Drive-by aligned `mega-menu-pane/edit.tsx` Edit function signature to the codebase convention `Edit(): Element` (see `locations-map/edit.tsx`, `navigation-menu/edit.tsx`). The prior `Edit(_props: BlockEditProps<BlockAttributes>)` was a pre-existing ESLint violation that prettier's line-collapse surfaced once the TEMPLATE change was applied. Same-file scope only; no unrelated files touched.
