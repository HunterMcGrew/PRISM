## Goal

Clarify build verification ownership across the agent persona spec so Clove doesn't claim a build step she doesn't run, and Briar runs the full build only when the diff actually exercises the Next.js bundle.

---

## Implementation Tasks

### Clove (implementation)

1. Update Clove DoD from "Build and type checks pass" to "Type checks pass" in `.claude/skills/thrive-code-dev/SKILL.md`.
2. Rewrite the Build step in `.claude/skills/thrive-code-review-self/SKILL.md` with conditional skip/run rules and the rationale paragraph (what build catches that tsc/tests don't).
3. Annotate step 5 in `.claude/rules/verification-commands.md` with Briar ownership, the conditional, and CI as the enforcement layer.
4. Update PR Readiness "Build passes" line in `.claude/rules/branch-plan.md` to support a "skipped — diff does not affect Next.js bundle" state.

---

## Decisions

- Briar owns the full `pnpm run build`. Clove stops at step 4 (tests). CI runs the build on PR open as the enforcement layer.
- Build is skipped on `.scss`/`.css`/`.md`/`.json`/`.claude/`-only diffs, backend-plugin-only diffs, test-only diffs, and Storybook-story-only diffs. Default to running when in doubt.
- Build is run on frontend `.tsx`/`.ts`, Next/Tailwind config, route segment files, layouts, middleware, `package.json`/`pnpm-lock.yaml`, and any `"use client"`/`"use server"` directive add/remove/move.
- The build's unique signal is RSC boundary leaks, `"use client"` directive issues, route-level Next compilation errors, and bundler-level circular dependency problems — that's why it stays in the workflow despite the cost, and why the conditional preserves the signal where it matters.
