## Acceptance Criteria

> Covers THR-1263 (columns + column blocks) and THR-1410 (alignment controls).

### Mega Menu Columns Block

- [ ] Navigate to a page with a Mega Menu block in the WordPress editor. Verify that a **Mega Menu Columns** block can be inserted inside a Mega Menu Pane.
- [ ] With a Mega Menu Columns block selected, open the Settings panel on the right. Verify a **Number of Columns** slider is present with a minimum of 1 and a maximum of 4.
- [ ] Increase the Number of Columns slider from 1 to 3. Verify that two additional column blocks appear inside the Mega Menu Columns block in the editor.
- [ ] Decrease the Number of Columns slider from 3 to 2. Verify that the rightmost column block is removed from the editor.
- [ ] With a Mega Menu Columns block selected, verify a **Stack on Mobile** toggle is present in the Settings panel.
- [ ] Enable the Stack on Mobile toggle and view the page on a mobile-sized viewport on the live frontend. Verify that the columns are displayed stacked vertically.
- [ ] Disable the Stack on Mobile toggle and view the page on a mobile-sized viewport on the live frontend. Verify that the columns are displayed side by side.
- [ ] With a Mega Menu Columns block selected, verify a **vertical alignment** toolbar control is available (top, center, bottom options).
- [ ] Set vertical alignment to **center** on the Mega Menu Columns block and view the live frontend. Verify that the content inside the columns is vertically centered.

### Mega Menu Column Block

- [ ] Inside a Mega Menu Columns block, verify that each child **Mega Menu Column** block can be individually selected.
- [ ] With a Mega Menu Column block selected, open the Settings panel and verify a **Width** field is present. Verify it accepts percentage values (e.g. "50%") and unit-based values (e.g. "200px").
- [ ] Set the width of one column to "33%" and leave a sibling column's width empty. View the live frontend and verify the first column occupies roughly one-third of the row while the second fills the remaining space.
- [ ] Clear the width field on a column (leave it empty). View the live frontend and verify the column automatically fills available space equally with its siblings.
- [ ] Verify that content blocks can be added inside a Mega Menu Column — including headings, paragraphs, images, buttons, and lists.

### Alignment Controls (THR-1410)

- [ ] With a Mega Menu Column block selected, verify a **horizontal alignment** toolbar appears with left, center, right, and stretch options.
- [ ] Set horizontal alignment to **center** on a column containing a heading. View the live frontend and verify the heading is horizontally centered within the column.
- [ ] Set horizontal alignment to **right** on a column. View the live frontend and verify the content is aligned to the right edge of the column.
- [ ] With a Mega Menu Column block selected, verify a **vertical alignment** toolbar appears with top, center, and bottom options.
- [ ] Set vertical alignment to **bottom** on a column. View the live frontend and verify the content sits at the bottom of the column.
- [ ] Change alignment settings on one column. Verify that sibling columns within the same Mega Menu Columns block are unaffected.
