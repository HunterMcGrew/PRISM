# Plan: THR-1531

## Ticket

https://linear.app/tractru/issue/THR-1531/add-remark-github-alerts-plugin-to-docs-site

## Goal

Add `remark-github-alerts` to the Nextra docs site so GitHub-flavored alert syntax renders as styled callouts in both GitHub and Nextra.

---

## User Stories

Not applicable — small improvement with clear scope.

---

## Decisions

- Use `remark-github-alerts` package (not Nextra's `<Callout>` component) — enables one markdown syntax that works on both GitHub and Nextra without `.mdx` conversion
- Import the plugin's shipped CSS (base + light + dark-class) rather than writing custom styles — matches GitHub's visual language, reduces maintenance
- Use `github-colors-dark-class.css` variant (not `-dark-media`) — Nextra theme-docs toggles dark mode via `.dark` class, not `prefers-color-scheme`

---

## Implementation Tasks

1. Install `remark-github-alerts` in `docs/` — run `cd docs && pnpm add remark-github-alerts`
2. Update `docs/next.config.mjs` — import `remarkGithubAlerts` and pass it to `nextra({ mdxOptions: { remarkPlugins: [remarkGithubAlerts] } })`
3. Add CSS imports in `docs/app/layout.jsx` — import `github-base.css`, `github-colors-light.css`, and `github-colors-dark-class.css` from the package, after the existing `nextra-theme-docs/style.css` import
4. Verify alerts render — check `go-live-procedure.md` and `dealer-site-spinup.md` in dev mode, both light and dark themes
5. Run production build — `cd docs && pnpm build` to confirm no build errors

---

## Acceptance Criteria

### Behavioral
- [ ] Given a doc page with `> [!WARNING]` syntax, When viewed on the Nextra docs site, Then the alert renders as a styled callout box with a warning icon and colored border — not as literal text
- [ ] Given a doc page with `> [!NOTE]` syntax, When viewed on the Nextra docs site, Then the alert renders as a styled callout box with a note icon and colored border
- [ ] Given the docs site is set to dark mode, When viewing an alert callout, Then the callout colors adapt to the dark theme
- [ ] Given the same markdown file viewed on GitHub, When viewing the alert syntax, Then GitHub renders its native alert styling (no change to GitHub rendering)

### Non-behavioral
- [ ] The docs site production build (`pnpm build`) completes without errors
- [ ] No custom CSS files are created except a minimal override for spacing issues not covered by the plugin's shipped stylesheets

### AC Adjustments

### AC Adjustment: Allow minimal custom CSS for margin fix
- **Original:** No custom CSS files are created — only the plugin's shipped stylesheets are imported
- **Proposed:** No custom CSS files are created except a minimal override for spacing issues not covered by the plugin's shipped stylesheets
- **Reason:** The plugin's `github-base.css` doesn't include enough top margin for Nextra's heading layout — alerts render flush against the heading above them. A single margin-top rule in `docs/app/custom.css` fixes this.
- **Status:** `accepted`
