# Plan: THR-1470

## Ticket

https://linear.app/tractru/issue/THR-1470/add-fetchpriorityhigh-to-hero-cards-lcp-image

## Goal

Add `fetchPriority="high"` and remove redundant `loading="eager"` from hero card Image components to improve LCP performance.

---

## Decisions

- `fetchPriority="high"` is a pass-through HTML attribute on `next/image`, separate from `priority` which only handles the preload `<link>` tag and `loading` behavior
- `loading="eager"` is redundant when `priority={true}` is set — Next.js sets it automatically

---

## Implementation Tasks

1. `frontend/components/hero-cards/HeroCards.tsx:109-117` — Remove `loading="eager"`, add `fetchPriority="high"` to the background `<Image>`
2. `frontend/components/card/HeroCardImage.tsx:23-32` — Remove `loading="eager"`, add `fetchPriority="high"` to the card `<Image>`
