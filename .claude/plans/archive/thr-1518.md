# Plan: THR-1518

## Ticket

https://linear.app/tractru/issue/THR-1518/fix-crash-on-promotion-detail-page-when-eligible-products-lack

## Goal

Fix runtime crash on promotion detail pages by aligning the eligible products query with `ProductBaseFields` and hardening the DTO type for WooCommerce null taxonomy behavior.

---

## Decisions

- Spread `...ProductBaseFields` in the eligible products fragment â€” aligns the promotion query with every other product query path. The hand-rolled inline fragment was the root cause: it drifted out of sync when `productTags` was added to base fields in PR #1645.
- Keep `productTags` as optional and nullable on `GQLProductDTO` â€” WooCommerce returns `null` for taxonomy connections on products with no tags, regardless of whether the field is queried.
- Keep optional chaining with fallback in the mapper (`dto.productTags?.nodes ?? []`) â€” belt and suspenders. The fragment fix prevents `undefined`; the type fix handles `null`.
- Do not introduce a lightweight eligible product mapper (Option C) â€” premature abstraction for a bug fix. If a product summary type is needed in the future, it should be driven by concrete demand across multiple consumers.
- Do not make all DTO fields defensive (Option A) â€” weakens the type contract globally when only one query was the outlier.
- Extracted `PRODUCT_BASE_FIELDS` into `product-base-fields.tsx` to avoid circular dependency â€” `product-fields.tsx` imports `PROMOTION_BASE_FIELDS` and `promotion-fields.ts` now needs `PRODUCT_BASE_FIELDS`. Re-exported from `product-fields.tsx` so existing importers are unaffected.
- Moved WooCommerce null taxonomy connection rule from `code-standards.md` to `frontend-services.md` â€” the architect context file is the more detailed, appropriate home for domain-specific DTO guidance.

Keep the inline `... on Product` fragment for `creditApplications` (detail-only field the template needs). GraphQL merges both selections.

2. **Keep DTO type fix** â€” `productTags` stays as `productTags?: { nodes: ... } | null` on `GQLProductDTO` (already done). This handles WooCommerce returning `null` for tagless products.

3. **Keep mapper safety** â€” `dto.productTags?.nodes ?? []` in `base-product.ts` stays (already done). Safety net for both `undefined` and `null`.

5. **Verify tests pass** â€” Run both `ProductService.test.ts` and `PromotionService.test.ts`. Both have coverage for `null`/`undefined` `productTags` (already added).

6. **Update architect context** â€” Add notes to `frontend-wp.md` (use base fragments for nested entities) and `frontend-services.md` (WooCommerce null taxonomy behavior).

## Review Issues

### File extraction may be unnecessary
- **Severity:** `minor`
- **Status:** `open`
- **File:** `frontend/lib/wp/fragments/product/product-base-fields.tsx`
- **Problem:** `PRODUCT_BASE_FIELDS` was extracted to a new file to avoid a circular dependency between `product-fields.tsx` and `promotion-fields.ts`. The circular dep is real â€” `product-fields.tsx` imports `PROMOTION_BASE_FIELDS` and `promotion-fields.ts` now needs `PRODUCT_BASE_FIELDS`. With CommonJS module resolution, this would cause `SINGLE_PRODUCT_FIELDS` to get `undefined` for `PROMOTION_BASE_FIELDS` depending on evaluation order. However, the user has flagged this as potentially unnecessary â€” deferring to their judgment on whether to keep the extraction or find a simpler approach.
- **Suggested fix:** User to decide â€” either keep the extraction (safe) or test importing directly from `product-fields.tsx` to verify whether webpack handles this specific circular case correctly.

### productTags should be optional and nullable
- **Severity:** `minor`
- **Status:** `open`
- **File:** `frontend/lib/data-access/gql-product/GQLProductDTO.ts:43`
- **Problem:** `productTags` is nullable (`| null`) but not optional (`?`). The plan decided on both. Runtime-safe because the mapper uses `?.`, but the type would be inaccurate if a future query path skips `ProductBaseFields`.
- **Suggested fix:** Change to `productTags?: { nodes: { id: string; name: string }[] } | null`
