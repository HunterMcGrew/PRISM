# Plan: Project Context for Skills Ecosystem

## Ticket

https://linear.app/tractru/issue/THR-1576/add-project-context-to-skills-ecosystem

## Goal

Make all skills automatically aware that they operate in the tractru/thrive project (Linear team: THR, GitHub org: tractru) so they never ask the user which project or team they're working in.

---

## User Stories

- As a developer who works across multiple projects, I want Nora to know I'm in Thrive without asking, so that ticket creation doesn't require an extra round-trip.
- As any developer using the skills ecosystem, I want all skills to have project context available, so that GitHub and Linear interactions use the correct identifiers by default.

---

## Decisions

- Project context goes in `skills-ecosystem.md`, not per-skill files — it's already loaded by every skill via the `"**"` wildcard in `manifest.json`. One source of truth, zero duplication.
- No per-skill MEMORY.md — there's no second use case for per-skill memory yet. Premature infrastructure for a 5-line fix.
- Claude auto memory is per-user and not shared across the team — project identity belongs in the repo, not in personal memory.
- Nora's "Determine team" step changes from "ask" to "use THR" with a reference back to `skills-ecosystem.md § Project Context`.

---

## Implementation Tasks

1. **Add `## Project Context` section to `skills-ecosystem.md`** — insert after the opening paragraph, before `## Skill Roster`. Contains repository name, Linear team prefix, and GitHub org. Includes a one-liner: "do not ask which team."
2. **Update Nora's `SKILL.md` step 4 (create-ticket path)** — change "ask which Linear team" to "use the THR team (see `skills-ecosystem.md § Project Context`). Do not ask."
