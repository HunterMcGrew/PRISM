## Decisions

- Criticals split out of THR-1564 into this story — minimal blast radius, fast ship. Reason: clearing Criticals quickly reduces dashboard signal noise and stakeholder anxiety, without entangling with the 20+ package sweep still pending for THR-1564.
- Use `pnpm.overrides` rather than forking or pinning direct deps — both handlebars and basic-ftp are transitive via dev-only parents (ts-jest, @puppeteer/browsers) that haven't released a bumped version. Override is the least-invasive path.
- Version specifiers use `^` rather than exact pin — allows future patch-level security fixes to flow without another override edit. Example: `"handlebars": "^4.7.9"` accepts `4.7.10`, `4.7.11`, etc.
- Comment pattern for override-to-alert mapping uses a sibling `"//"` field inside the `pnpm` object. Reason: JSON forbids inline comments, and a sibling field is idiomatic for pnpm config documentation. THR-1564 will extend this same pattern.
- This story establishes the `pnpm.overrides` pattern in root `package.json` — THR-1564 inherits and extends it rather than introducing the pattern later.
- Dev-only "Critical" alerts are patched, not dismissed — the fix is trivial and dismissing sets a bad precedent.
