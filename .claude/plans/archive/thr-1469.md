# Plan: THR-1469

## Ticket

https://linear.app/tractru/issue/THR-1469/fix-modal-dialog-accessibility-empty-heading-missing-dialog-name-close

## Goal

Fix three WCAG 2.1 AA accessibility failures in the popup modal: empty heading, missing dialog name, and inaccessible close button.

---

## Decisions

- Conditionally render `DialogTitle` only when `formTitle` is truthy ŌĆö prevents empty `<h3>` in DOM
- When `formTitle` is absent, add `aria-label` on the `Dialog` using `DEFAULT_FORM_TITLE` ŌĆö avoids a new magic string, ensures dialog always has an accessible name
- Wrap close icon in `<button type="button" aria-label="Close dialog">` ŌĆö `type="button"` prevents accidental form submission in form-wrapping consumers (e.g. SharedModals)
- Rename Props type to `ModalDialogProps` ŌĆö aligns with `{ComponentName}Props` convention while we're in the file

---

## Implementation Tasks

### Task 1: Fix ModalDialog accessibility (ModalDialog.tsx)

**File:** `frontend/components/utility/ModalDialog.tsx`

3. Add `aria-label={DEFAULT_FORM_TITLE}` to `Dialog` when `formTitle` is absent ŌĆö Headless UI auto-wires `aria-labelledby` when `DialogTitle` is present, so the two won't conflict
4. Wrap `IoCloseCircleOutline` in `<button type="button" aria-label="Close dialog">` ŌĆö move `onClick={onClose}` to the button, reset default button styles to preserve current appearance
5. Verify no changes needed in `ModalBlock.tsx` or other consumers (the interface is unchanged ŌĆö `formTitle` remains optional, close behavior unchanged)
