# Plan: thr-1777

## Ticket

https://linear.app/tractru/issue/THR-1777/fix-workspace-clean-scripts-and-add-deps-cleanup

## Goal

Replace the broken root `clean` script and add a workspace-wide `clean:*` namespace for cleaning build artifacts and `node_modules`.

---

## Decisions

- Orchestration mirrors the existing `check-types` pattern (`pnpm -r run <script>`) — each package owns its own artifact knowledge; root just fans out.
- Root `clean:deps` removes the root `node_modules` *after* the recursive sweep — pnpm needs the root `node_modules/.pnpm` intact while iterating workspaces.
- `clean:reset` chains `clean && pnpm install` with `&&` so a failed clean aborts the install.
- Scope is intentionally limited to `frontend` and `backend/plugins/gravity-platform-core` — `docs` and `packages/ci-scripts` are out of scope. The `pnpm -r` orchestration silently skips members without the script, which is the desired behavior.

## Acceptance Criteria

### Behavioral

- [ ] Given the repo is fresh from `origin/main`, When the user runs `pnpm clean:build` from the repo root, Then `.next/.turbo` is removed from `frontend/` and `build/.turbo` is removed from `backend/plugins/gravity-platform-core/`.
- [ ] Given the repo is fresh from `origin/main`, When the user runs `pnpm clean:deps` from the repo root, Then `node_modules` is removed from the root, `frontend/`, and `backend/plugins/gravity-platform-core/`.
- [ ] Given the user runs `pnpm clean`, When it completes, Then both targeted build artifacts and the targeted `node_modules` directories are gone.
- [ ] Given the user runs `pnpm clean:reset`, When it completes, Then a fresh `pnpm install` has run successfully and `pnpm dev` and `pnpm check-types` succeed afterward.
- [ ] Given the user runs `pnpm clean:build` from inside `frontend/`, When it completes, Then only `frontend/.next` and `frontend/.turbo` are removed.
- [ ] Given the user runs `pnpm clean:deps` from inside `frontend/`, When it completes, Then only `frontend/node_modules` is removed.
