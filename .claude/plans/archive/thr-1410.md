# Plan: THR-1410

## Ticket

THR-1410

## Goal

Add horizontal and vertical alignment toolbar controls to mega-menu-column and mega-menu-columns blocks.

---

## Decisions

- `BLOCK_ALIGNMENT_MAPPINGS` and `BLOCK_VERTICAL_ALIGNMENT_MAPPINGS` are the preferred constants for WordPress toolbar ‚Üí Tailwind mapping in column-direction flex containers. The old `CROSS_AXIS_ALIGN_MAPPINGS` / `MAIN_AXIS_ALIGN_MAPPINGS` remain for direction-agnostic contexts (group, carousel) until migrated.
- Mega-menu column/columns attributes are defined in `index.tsx`, not `block.json` ‚Äî consistent with project convention.
- Category for mega-menu blocks is `"gravity-platform"`, not `"layout"`.
- Mega-menu-column reuses `ColumnBlockAttributes` from `frontend/blocks/columns/schema` rather than defining its own type ‚Äî keeps alignment types in one place.
- Follow-up needed: migrate remaining consumers of `CROSS_AXIS_ALIGN_MAPPINGS` / `MAIN_AXIS_ALIGN_MAPPINGS` to the new constants, then remove the old ones.

## Debugged Issues

### ColumnBlock props require full BaseBlockProps ‚Äî tests fail type-check
- **Status:** `fixed`
- **File:** `frontend/blocks/columns/ColumnBlock.tsx:19`
- **Root cause:** `BlockFrontendProps = Partial<BlockAttributes> & BaseBlockProps` requires all 6 `BaseBlockProps` fields (`id`, `blockIndex`, `blockName`, `innerHTML`, `isInnerBlock`, `isLastBlock`), but `ColumnBlock` only uses `id` from that set ‚Äî it's called both via `BlocksRenderer` (which supplies all fields) and directly (e.g. mega-menu-column, tests) without them.
- **Recommended fix:** Change line 19 to `type BlockFrontendProps = Partial<BlockAttributes> & Partial<BaseBlockProps>;` ‚Äî makes the pipeline fields optional to match actual component usage.
- **Suggested tests:** Existing tests cover this; they will pass once the type is corrected. No new tests needed.
- **Fixed in:** Changed `BlockFrontendProps` to `Partial<BlockAttributes> & Partial<BaseBlockProps>` in `ColumnBlock.tsx:19`.

### Offer.stories.tsx smoke-test timeout ‚Äî missing `site` prop
- **Status:** `fixed`
- **File:** `frontend/single-templates/Offer.stories.tsx:244`
- **Root cause:** Story args omit `site: ISiteGlobals`, so `Offer` receives `site = undefined`, causing `ServiceFactory.createCallToActionService(undefined)` to throw `TypeError: Cannot read properties of undefined (reading 'wpUrl')` before any mock is reached ‚Äî the async component never resolves and the test runner times out.
- **Recommended fix:** Add `site` to the story args: `site: getMockSiteGlobals()` ‚Äî the variable is already in scope at line 12. Also remove the irrelevant `fetchSiteGlobals` mock (the component doesn't call it).
- **Suggested tests:** none needed ‚Äî the smoke-test itself validates this once the fix is in.
- **Fixed in:** Added `site` to Test story args; removed dead `fetchSiteGlobals` mock and its import from `Offer.stories.tsx`.
