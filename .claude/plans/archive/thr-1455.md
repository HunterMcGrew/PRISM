# Plan: THR-1455

## Ticket

THR-1455 — RSC-first form props and Apollo cleanup (follow-up to THR-1454)

## Goal

Remove client-side form fetches (`useGetGravityForm` / `/api/form`) from Kronos Search, Equipment List, and the main contact flow by fetching forms server-side and passing them as props.

---

## Decisions

- `quoteForm` resolved in `kronos-search/resolver.props.ts` — shared with editor, so the form is pre-fetched for both server render and editor preview.
- `ctaForm` resolved in a new `equipment-list/resolver.props.ts` — same pattern as Kronos; the `ctaFormId` block attribute drives the fetch.
- `mainContactForm` added to `AppContextPageProps` and fetched in `app/layout.tsx` — the contact form is site-wide and known at render time, so it belongs at the layout level.
- `useContactCTAs` passes `form: mainContactForm` directly to `openModal` instead of `formId: mainContactFormId` — eliminates the client fetch triggered by `SharedModals` for the header contact button.
- `useGetGravityForm` / `/api/form` kept for button blocks that open arbitrary form IDs — cannot be pre-fetched at build time; this is the remaining isolated usage.
- `useCTAProps` `useGetGravityForm` usage deferred — requires changes to CTA data flow across many blocks; tracked as follow-up.
