## Decisions

- The `MEGA_MENU_ALLOWED_BLOCKS` constant contains 14 entries total: 7 mega-menu-prefixed content/structural blocks (`mega-menu-about-us`, `mega-menu-columns`, `mega-menu-featured-banner`, `mega-menu-featured-links`, `mega-menu-info-banner`, `mega-menu-links-with-icon`, `mega-menu-locations`, `mega-menu-navigation-list`) plus 6 core/card blocks (`core/buttons`, `core/heading`, `core/image`, `core/list`, `core/paragraph`, `gravity-platform-core/card`). Docs that say "14 block types" and then list only the mega-menu-specific blocks are misleading.
- The container block is named `mega-menu-links-with-icon` (singular) in the source. Its inner block is `mega-menu-link-with-icon` (also singular). Docs that say "links-with-icons" are wrong.
- `closeMenu()` and `closeDropdown()` on `MegaMenuContext` are aliases for the same `safeCloseMenu` callback â€” documenting them as two distinct methods with different semantics is incorrect.
- Migration dismissal on current main is server-persisted: the `platform_settings_mega_menu_migration_dismissed` option is set via `PUT /thrive-mega-menu/{id}/migrate/dismiss` and returned as `dismissed` on `GET /migrate/status`. THR-1625 (shipped) hides the "Don't show again" checkbox when the modal is opened manually via the Import button. THR-1626 is NOT on main yet â€” it reworks dismissal to per-user localStorage and removes the endpoint + option + `dismissed` status field, and it ships its own doc updates. THR-1627 documents the current shipped state; the post-THR-1626 audit is a separate follow-up.
- `FeatureFlags` lives at `includes/Settings/FeatureFlags.php`, not `includes/Services/`. The architect doc file path is stale.
