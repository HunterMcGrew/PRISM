# Plan: THR-1575

## Ticket

https://linear.app/tractru/issue/THR-1575/gpc-opt-out-notice-toast

## Goal

Display a passive GPC opt-out notice toast when the browser's Global Privacy Control signal is detected, as a legal compliance requirement.

---

## User Stories

### Story 1: GPC detection notice
**As a** site visitor with Global Privacy Control enabled in my browser,
**I want to** see a brief notice confirming my privacy preference has been detected,
**so that** I know the site is honoring my opt-out without needing to take any action.

**Acceptance criteria hints:**
- [ ] Given GPC is enabled (`navigator.globalPrivacyControl === true`), When the user visits any page, Then a toast appears in the top-right corner with a green checkmark, "Privacy preference honored" heading, and explanatory text
- [ ] Given GPC is not enabled, When the user visits any page, Then no toast is shown
- [ ] Toast is positioned top-right, 16px from edges on desktop; full-width on mobile

### Story 2: Toast dismissal
**As a** site visitor who has seen the GPC notice,
**I want to** dismiss it manually or have it disappear on its own,
**so that** it doesn't get in the way of browsing.

**Acceptance criteria hints:**
- [ ] Given the toast is visible, When 7 seconds elapse, Then the toast auto-dismisses
- [ ] Given the toast is visible, When the user clicks the × button, Then the toast dismisses immediately
- [ ] Given the toast is visible, When the user presses Escape, Then the toast dismisses

### Story 3: Session persistence
**As a** site visitor with GPC enabled,
**I want to** see the notice only once per browsing session,
**so that** I'm not repeatedly notified on every page.

**Acceptance criteria hints:**
- [ ] Given the toast has already been shown this session, When the user navigates to another page, Then the toast does not reappear
- [ ] Given the user closes and reopens the browser, When they visit the site again, Then the toast shows again (new session)

### Story 4: Accessible notice
**As a** site visitor using a screen reader or keyboard navigation,
**I want to** be informed of the GPC notice and able to dismiss it,
**so that** the privacy acknowledgment is equally accessible to me.

**Acceptance criteria hints:**
- [ ] Toast region has appropriate ARIA attributes (`role="status"` or `aria-live="polite"`)
- [ ] Dismiss button is keyboard-focusable and has an accessible label
- [ ] Toast does not trap focus — it's passive, not a dialog

## Decisions

- Client component (`"use client"`) — `navigator` and `sessionStorage` are browser-only APIs, RSC is not possible
- Termly is separate — this notice is passive acknowledgment only, not a consent mechanism
- Tailwind exclusively — no inline styles (unlike the WP reference implementation)
- Session-scoped visibility — `sessionStorage` gate, not `localStorage` (resets per session, matching the WP behavior)
- Position top-right (not bottom-left) — avoids potential overlap with Termly's consent banner which renders bottom-left
- Full-width on mobile — `25vw` is unreadable on small viewports; use `left-4 right-4` on mobile, `sm:w-[25vw] sm:max-w-xs` on desktop
- Hydration via `useState(false)` + detection effect — `navigator` doesn't exist during SSR, so lazy initializer would cause hydration mismatch. Effect pattern gives identical SSR and initial client render. Matches TermlyCMP pattern in this codebase.
- `z-50` (Tailwind built-in, z-index: 50) — above page content, below chat widgets (z-2147483000) and modals
- Component lives in `frontend/components/gpc-opt-out-notice/` — domain-specific to privacy compliance, not a generic utility. Own directory because it needs a story file.
- Not a generic toast — this is purpose-built for GPC compliance. If the project needs a toast system later, build it then. No premature abstraction.
- Fade-out on dismiss only — no fade-in (toast appears instantly for compliance clarity). Dismiss triggers a 300ms CSS opacity transition before unmount. `motion-reduce:duration-0` respects `prefers-reduced-motion`.

## Implementation Tasks

1. **Create `GpcOptOutNotice` component** — `frontend/components/gpc-opt-out-notice/GpcOptOutNotice.tsx`
   - `"use client"` directive
   - `useState(false)` for visibility (SSR-safe initial value)
   - Detection effect (empty deps): check `navigator.globalPrivacyControl === true` and `sessionStorage.getItem('gpc_notice_shown')`. If both pass, write `sessionStorage.setItem('gpc_notice_shown', '1')` and `setVisible(true)`.
   - Auto-dismiss effect (deps: `[visible]`): `setTimeout` 7000ms → `setVisible(false)`. Cleanup: `clearTimeout`. Required per useEffect guidelines.
   - Escape key effect (deps: `[visible]`): `document.addEventListener('keydown', ...)`. Cleanup: `removeEventListener`. Only active when visible.
   - JSX: fixed-position card with green checkmark SVG, heading, body text, dismiss button. Tailwind only.
   - Responsive: `fixed top-4 left-4 right-4 sm:left-auto sm:right-4 sm:w-[25vw] sm:max-w-xs`
   - `z-50` for stacking context
   - `role="status"` on container, `aria-label="Dismiss privacy notice"` on button
   - Return `null` when not visible
   - Props type: `GpcOptOutNoticeProps` (empty interface, or no props — component is self-contained)

2. **Add to root layout** — `frontend/app/layout.tsx`
   - Import `GpcOptOutNotice`
   - Place `<GpcOptOutNotice />` in the `<body>` near `TermlyCMP` (after it, before the body scripts)
   - No props needed — component is self-contained

3. **Write unit tests** — `frontend/components/gpc-opt-out-notice/__tests__/GpcOptOutNotice.test.tsx`
   - Mock `navigator.globalPrivacyControl` and `sessionStorage`
   - Test: renders toast when GPC is enabled and session gate is clear
   - Test: does not render when GPC is disabled
   - Test: does not render when session gate is set
   - Test: sets sessionStorage on mount
   - Test: dismiss button removes the toast
   - Test: auto-dismisses after 7 seconds (fake timers)
   - Test: Escape key dismisses the toast
   - Test: has `role="status"` and accessible dismiss button
   - Depends on: Task 1

4. **Write Storybook story** — `frontend/components/gpc-opt-out-notice/GpcOptOutNotice.stories.tsx`
   - Default story showing the toast visible (mock the browser APIs so it renders)
   - Consider using a wrapper that forces `visible` state for visual testing
   - Depends on: Task 1

## Review Issues

### Unguarded sessionStorage access
- **Severity:** `minor`
- **Status:** `fixed`
- **File:** `frontend/components/gpc-opt-out-notice/GpcOptOutNotice.tsx:19-27`
- **Problem:** `sessionStorage.getItem()`/`setItem()` can throw `SecurityError` in restricted contexts. Component mounts at root layout level outside any error boundary — uncaught throw crashes the entire client tree.
- **Suggested fix:** Wrap the detection effect body in `try/catch`. On error, fail silently (skip the notice).
- **Fixed in:** Added try/catch around detection effect. Test added for SecurityError case.

## Acceptance Criteria

### Behavioral
- [ ] Given a browser with GPC enabled, When the user visits any page, Then a toast appears in the top-right corner displaying a green checkmark icon, "Privacy preference honored" heading, and the message "Your Global Privacy Control opt-out has been detected. We are not selling or sharing your data."
- [ ] Given a browser without GPC enabled, When the user visits any page, Then no toast is shown
- [ ] Given the toast is visible, When 7 seconds elapse, Then the toast disappears automatically
- [ ] Given the toast is visible, When the user clicks the × button, Then the toast disappears immediately
- [ ] Given the toast is visible, When the user presses the Escape key, Then the toast disappears immediately
- [ ] Given the toast has already appeared during this browser session, When the user navigates to another page or refreshes, Then the toast does not reappear
- [ ] Given the user closes the browser and opens it again, When they visit the site, Then the toast appears again (new session)
- [ ] Given a desktop viewport, When the toast is visible, Then it appears in the top-right corner with approximately 25% viewport width (max 320px)
- [ ] Given a mobile viewport, When the toast is visible, Then it spans the full width of the screen with small margins on each side

### Non-behavioral
- [ ] The toast has `role="status"` so screen readers announce its content without interrupting the user
- [ ] The dismiss button has an accessible label (not just the × character)
- [ ] The dismiss button is keyboard-focusable
- [ ] The toast does not trap focus — it is a passive notification, not a dialog
- [ ] Styled entirely with Tailwind classes — no inline styles
