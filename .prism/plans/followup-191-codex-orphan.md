# Plan: followup-191-codex-orphan

> Closed: 2026-06-16

## Ticket

GitHub issue #191 (follow-up to #164/PR #174)

## Goal

Delete `.generated/codex-config.toml`, a stale tracked file left over from the pre-ADR-0044 build layout.

---

## Decisions

- **`.generated/codex-config.toml` is an orphan — deleted with no documentation change needed.**
  - **Root cause:** ADR-0044 (PR #154) migrated `codexConfigFile` in `.ai-skills/definitions/paths.json` from `.generated/codex-config.toml` to `.codex/codex-config.toml`. `build.ts:1133-1135` now writes to `.codex/` via `pathDefinitions.generated.codexConfigFile`. The old `.generated/` file was never cleaned up post-migration (added in PR #102).
  - **Alternatives considered:** keep the file as reference output; redirect build to also write `.generated/`. Both rejected — the file has zero runtime consumers and duplicating build output adds noise without value.
  - **Chosen approach:** `git rm` the file. `.generated/` directory becomes empty and is removed. `.codex/codex-config.toml` is gitignored (`.gitignore` line 9) and regenerated by `pnpm prism:build` on each consumer's machine.
  - **Verification:** `grep -rn "codex-config" scripts/ .github/ package.json .ai-skills/definitions/paths.json` shows `build.ts` references `.codex/codex-config.toml` only (via `paths.json`). No CI workflows or build-time consumers reference `.generated/`. ADR-0044 already documents the migration — no new documentation needed.
  - **Sibling precedent:** #164/PR #174 deleted `.generated/cursor-skills/` as the same class of stale post-reorg orphan.
  - → no promotion needed (ticket-tactical deletion; ADR-0044 is the durable record)
