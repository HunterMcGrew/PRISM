## Goal

Add an executed AC-verification eval layer to PRISM: Reese gains an **executed AC Verification mode** (per-criterion verdicts with typed evidence, graded against the branch diff), Reese's verification enters Sol's lifecycle chain **before** the review loop, and Winston's acceptance criteria gain a **gradeability bar** (every criterion carries a stable ID and a falsifiable Evidence source tagged machine or human). Together these close the gap between deterministic ratification (exit codes) and freeform review (prose findings): an independent judge grading finished work against the plan's own AC.

**Why:** self-evaluation is the weak link — an independent evaluator grading against binary, evidence-grounded criteria is the mechanism with published gains behind it. Sol today accepts a write-lane `done` on deterministic ratification (re-run exit code) plus the doer's self-assessed confidence. The `## Acceptance Criteria` already written into every plan is a rubric nothing consumes; this layer makes it the grading instrument.

## The verdict contract (design core — read before any task)

Inherited verbatim from the locked portable design. Every task below conforms to it. The **routing verbs** are re-expressed in PRISM's native seven-signal set (see `## Decisions` → "PRISM-native routing" for why this differs from the portable `done`-with-field routing).

- **`acVerdicts` shape** (owned by `.prism/skills/prism-conductor/lib/report-back.md`; everyone else points at it): `[{id, criterion, verdict: MET|UNMET|UNGRADEABLE, evidenceType: executed|inspected|demonstrated, evidence, reason?}]`. `id` is the stable criterion ID (AC-1, AC-2…) assigned at authoring. `reason` is required on UNGRADEABLE: `ac-defect | harness | dead-reference | requires-human | converted`.
- **Reese's primary verdict maps to PRISM's review-rung enum** (the PRISM adaptation of the portable "`done`-whenever-verification-ran" rule):
  - all machine criteria MET (born-UNGRADEABLE side-findings permitted) → **`done`** (advance).
  - ≥1 UNMET → **`needs-fix`** — routes to the implementer (Clove) for the recorded `## Review Issues`, re-dispatches Reese; the gauntlet loop, bounded by the step-07 pass budget + three-strike rule. This *is* PRISM's native review-rung route (Briar/Eric return `needs-fix` on findings); the portable roster used `done`-with-field only because it lacked a `needs-fix` verdict.
  - all criteria UNGRADEABLE → **`needs-replan`** (→ Winston). Zero criteria were verified; a "done" here would advance a lane verified by no one.
  - no `## Acceptance Criteria` section at all → **`blocked`** (→ Winston).
- **Sol routes on the primary verdict — never re-judges a criterion.** This is the exit-code rule's sibling: Sol applies the `lib/report-back.md` routing table to Reese's returned verdict exactly as it does for Briar/Eric. The per-criterion detail rides the `acVerdicts` evidence field and the `## Review Issues` open entries — Clove reads *which* criteria to fix; Sol never parses per-criterion.
- **Verdict-state taxonomy** (inherited):
  - **Harness failure ≠ UNMET.** An evidence command that errors, can't run in the worktree, or disagrees with itself across two runs (flake) is UNGRADEABLE(`harness`) with the error captured — never UNMET, never `blocked` (unless the whole AC section is unreadable). A signal that can't run is not a failing signal.
  - **Dead references.** An evidence source naming a command/path that no longer resolves is UNGRADEABLE(`dead-reference`).
  - **Human-only criteria** (visual, timing, feel) are not AC defects. Winston tags each Evidence sub-bullet `machine` or `human` at authoring; Reese grades the machine set and emits the human set as a mini-checklist attached to the report — each criterion routed to the verifier that can actually verify it.
  - **Born vs converted UNGRADEABLE.** *Born* (vague/unfalsifiable/missing evidence) → recorded as a `## Review Issues` open entry, the lane **advances** on the gradeable set. *Converted* (a criterion that survived two Reese⇄Clove fix cycles) → `needs-replan` (→ Winston for criterion rewrite); a twice-failed criterion must not ride the side-finding channel past review and the merge gate as a cosmetic note.
  - **Born-UNGRADEABLE does NOT ride `found-followup-work`.** In PRISM `found-followup-work` routes to Nora (scope-fit + DoR → potentially a new ticket). A vague/dead AC criterion is a defect in the *current* plan's AC, owned by Winston — not new-ticket work. It rides a `## Review Issues` open entry (durable bus); Winston's closing-ceremony loose-thread check and Zoe's per-plan audit age it. An `observation` secondary signal may additionally surface it in the end-of-run report.
- **UNGRADEABLE(requires-human)** → emitted in the report's awaiting-human-verification mini-checklist, surfaced at the human merge gate (the unconditional gate in `lib/report-back.md` § Gate registry). Never graded, never silently dropped.
- **Disputed UNMET.** Clove disputing a verdict returns **`needs-replan`** quoting both readings — never an appeasement fix (a code change with no requirement behind it). It routes to Winston (the criterion's owner) via the step-06 disagreement fast-path; Winston sharpens the criterion or evidence source; Reese re-grades against the corrected version. Two competent readers reaching opposite verdicts is, by the design's own standard, an ambiguous criterion.
- **No per-criterion confidence grades.** A confidence dial is the partial-credit escape hatch binary grading exists to remove. Evidence is *typed* instead (`executed` re-runnable > `inspected` file-state > `demonstrated` self-reported); the type ratio is itself a signal — wall-to-wall `demonstrated` METs is the rubber-stamp tell.

## Decisions

### Inherited from the locked portable design (attribute lineage)

- **Reese owns executed AC verification; no new persona.** QA is where every contributor looks for AC verification; his bug-fix mode is the nearest shape. Considered and rejected by the portable panel: a dedicated verifier persona (roster bloat); folding into Briar (self-review is the *least independent* seat — putting the grading instrument there undermines the independent-judge premise). → no promotion needed (ticket-tactical; the durable form is Reese's mode spec).
- **Per-criterion verdicts are binary; UNGRADEABLE carries a required reason** (`ac-defect | harness | dead-reference | requires-human | converted`). Harness failures/flakes are never UNMET (a signal that can't run is not a failing signal); dead references are named, not judgment calls; human-only criteria route to the merge gate as a checklist; converted (twice-failed) pauses via `needs-replan`. → promoted to `.prism/skills/prism-conductor/lib/report-back.md` at close (C1 makes it live).
- **No per-criterion confidence grades; evidence is typed instead** (`executed | inspected | demonstrated`). Confidence dials reopen the partial-credit door binary grading closed. The type ratio is the rubber-stamp telemetry. → promoted to report-back.md § Evidence fields at close.
- **Gradeability bar requires falsifiable, not runnable:** procedure + expected observation + failure signature, positive controls on absence-evidence, behavioral evidence for behavioral criteria, machine/human tag, stable IDs. If Winston can't name what UNMET looks like, the criterion isn't ready. → promoted to `.prism/templates/acceptance-criteria.md` (A1 makes it live).
- **Disputed UNMET → `needs-replan` with both readings; Winston arbitrates** (step-06 disagreement fast-path). Never an appeasement fix. Two competent readers disagreeing *is* the definition of an ambiguous criterion. → no promotion needed (expressed in the routing contract + Clove/Reese bodies).
- **Contract shapes have one owner.** report-back.md owns the `acVerdicts` schema; Reese owns verdict semantics; Sol quotes only routing predicates; Winston (via the template) owns the Evidence format. The portable roster's own history is the evidence: 24 enum-quoting files needed a same-day harmonization audit. → promoted to report-back.md at close.
- **Eric samples the checker.** His spec subagent consumes the report, samples `demonstrated`-class METs first, re-executes `executed`-class citations; a flipped MET triggers full re-grade at top tier (miscalibrated judge, not a point defect). He is the only checker of the checker. → no promotion needed (Eric-body behavior).
- **Reese verifies before the review loop.** UNMET caught pre-review costs one Clove dispatch; post-review it costs Briar + Eric twice. → promoted to `step-04-dispatch.md` chain (C2).
- **Back catalog:** criterion present, Evidence absent → Reese derives the obvious evidence labeled `(derived)`, UNGRADEABLE only when he can't. Prevents a day-one flood of side-findings that trains everyone to ignore the channel. → no promotion needed (Reese mode spec).

### PRISM-forced decisions (the portable plan did not face these)

- **PRISM-native routing: Reese's any-UNMET primary verdict is `needs-fix`, not the portable `done`-with-`acVerdicts`.** PRISM's report-back contract already has a `needs-fix` review-rung verdict (Briar/Eric return it on findings) that routes exactly to the Clove-fix → re-review gauntlet loop, bounded by the step-07 pass budget + three-strike rule. The portable roster used `done`-with-field *only because it lacked a `needs-fix` verdict* ("mirroring the review-loop precedent where reviewers return `done` with findings") — that precedent is portable-specific; PRISM's review-loop precedent is `needs-fix`. The portable invariant ("Sol routes deterministic predicates, never re-judges a criterion") is **preserved and strengthened**: Sol routes on the primary verdict (its native deterministic mode), and `acVerdicts` + `## Review Issues` carry the per-criterion detail to Clove. **Alternative considered:** import the portable `done`-with-field routing verbatim. **Rejected:** it would fork a parallel routing path alongside PRISM's existing `needs-fix` loop, duplicate the gauntlet machinery, and force Sol to parse a per-criterion field — the exact "route a verdict, never interpret one" violation the seven-signal set exists to prevent. → promoted to report-back.md § Routing table (C1) at close.
- **Born-UNGRADEABLE(ac-defect/dead-reference/harness) does NOT ride `found-followup-work`.** In PRISM `found-followup-work` has a specific route — Nora's scope-fit + DoR gate, potentially a new ticket. A vague/dead/un-runnable AC criterion is a defect in the *current* plan's AC, owned by Winston, not new-ticket work. It rides a `## Review Issues` open entry (durable bus) + `observation` signal; the lane advances on the gradeable set; Winston's closing-ceremony loose-thread check and Zoe's audit age it. **Why it matters:** a naive signal mapping would send AC defects to Nora, who would either file spurious tickets or bounce them — noise that trains the team to ignore the channel. → promoted to report-back.md § Secondary signals (C1) at close.
- **The portable single-file `sol/SKILL.md` Task 3 explodes across PRISM's step/lib files.** Chain → `step-04-dispatch.md` (write-phase chain) + `lib/goal-state.md` (phase enum); routing predicates + shape → `lib/report-back.md` (the canonical routing table + evidence-field owner), cited by `step-05-route.md`; model tier → `shared.md § Model tiering`; read-phase `agent()` schema → `claude.md`. Budgets need **no** edit — the loop rides the existing `needs-fix` gauntlet already bounded by `step-07`. → no promotion needed (structural mapping recorded here for the implementer).
- **Shape owner is `lib/report-back.md`, not a `core.md`.** PRISM has no `_shared/core.md`; the report-back contract doc is the natural single owner of the `acVerdicts` evidence-field schema (it already owns the verdict enum, evidence fields, routing table, and gate registry, with step files citing it). → promoted (C1) at close.
- **Build-mirror parity is a first-class task.** PRISM skill files are build-generated: `.ai-skills/skills/<persona>/shared.md` (+ `frontmatter.yml`/`claude.md`) → `.claude/skills/<persona>/SKILL.md`, and `.codex/agents/*.toml` + `.claude/agents/*.md` are also generated. The portable roster's hand-edited `codex-agents/reese.toml` maps in PRISM to editing `frontmatter.yml` + regenerate — the toml is a mirror, never hand-edited. Task E2 runs `pnpm prism:build` + `pnpm prism:verify-pack` + `pnpm prism:check` as the parity gate. Conductor `.prism/skills/prism-conductor/{step,lib}/*.md` and Iris's `.prism/skills/prism-retro/step-*.md` are hand-authored canonical (runtime-read, not regenerated) — edited directly. → no promotion needed (build discipline; already in `.prism/rules/skill-authoring.md`).
- **Mode externalization: the AC-verification procedure is a reference, not inlined in `shared.md`.** PRISM's disclosure model (`.prism/rules/skill-authoring.md`, 500-line body cap) externalizes each Reese mode to `.prism/references/qa-test-plan/mode-*.md`. The portable roster inlined the mode body. - **`ac-verify` is a NEW phase, not the existing `qa` phase.** PRISM's step-04 chain already carries a post-review `qa` phase (Reese's tester-facing checklist mode). AC Verification is a *separate* phase inserted **before** self-review: `implement → ac-verify → self-review → pr-review → qa → docs`.
