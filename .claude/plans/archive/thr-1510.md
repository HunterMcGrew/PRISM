# Plan: THR-1510

## Ticket

https://linear.app/tractru/issue/THR-1510/clarify-reorder-vs-navigation-labels-on-dealer-announcement-carousel

## Goal

Clarify the slide reorder controls in the Dealer Announcement Carousel editor so users don't confuse them with slide navigation.

---

## User Stories

- As a content author editing a Dealer Announcement Carousel, I want the reorder buttons to clearly indicate they change slide position, so that I don't accidentally reorder slides when I meant to navigate to a different one.

---

## Decisions

- Use "Reorder Slide" as the BaseControl label instead of a full instructional sentence — short noun phrase is unambiguous and matches the structural nature of the action
- Use `help` prop on BaseControl for guidance text — follows the established pattern documented in `backend-editor-components.md`
- Button labels use "Shift Earlier" / "Shift Later" instead of "Move Slide Left" / "Move Slide Right" — "shift" implies repositioning, "earlier/later" frames as sequence position rather than spatial direction, avoids overlap with carousel arrow navigation
- Helper text explicitly mentions carousel arrows — directly addresses the observed confusion by pointing users to the right control

---

## Implementation Tasks

1. **Relabel BaseControl** — In `backend/.../dealer-announcement-carousel/edit.tsx` line 216, change the `label` prop from `"Move the slide left or right in the slide order."` to `"Reorder Slide"`. Add `help="Changes this slide's position in the carousel. Use the carousel arrows to switch slides."`
2. **Relabel buttons** — Change "Move Slide Left" (line 225) to "Shift Earlier" and "Move Slide Right" (line 236) to "Shift Later"
3. **Verify in editor** — Build the plugin (`pnpm run build` in backend), open a page with the carousel block, confirm labels render correctly and reorder behavior is unchanged

## Acceptance Criteria

### Behavioral
- [ ] Given a Dealer Announcement Carousel block is selected in the editor, When the sidebar panel opens, Then the reorder section heading reads "Reorder Slide"
- [ ] Given the reorder section is visible, When the user reads the helper text, Then it reads "Changes this slide's position in the carousel. Use the carousel arrows to switch slides."
- [ ] Given the reorder section is visible, When the user reads the button labels, Then the left button reads "Shift Earlier" and the right button reads "Shift Later"
- [ ] Given a carousel with 3+ slides and the middle slide is selected, When the user clicks "Shift Earlier", Then the slide moves one position earlier in the sequence and the sidebar still shows that slide's content
- [ ] Given the first slide is selected, When the user views the reorder buttons, Then "Shift Earlier" is disabled
- [ ] Given the last slide is selected, When the user views the reorder buttons, Then "Shift Later" is disabled
- [ ] Given a carousel with multiple slides, When the user clicks the carousel arrows in the preview, Then the active slide switches for editing (unchanged behavior)

- [ ] Existing slide reordering logic unchanged — only label text modified
