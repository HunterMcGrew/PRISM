# Branch Plan: feature/hmcgrew/thr-1445-sortable-links-modal

## Goal

Introduce a reusable `SortableLinksModal` editor component and update `LinkControl` / `ToolbarLink` to accept `LinkValue` (`string | ILink`) instead of `ILink` only.

---

## Decisions

- `LinkValue = string | ILink` accepted by `LinkControl`, `ToolbarLink`, so existing blocks that store a URL string can use these components without migrating their attributes.
- `ensureLinkObject` wraps string values at block-attribute write time — callers (edit.tsx files) are responsible for normalizing before calling `setAttributes`.
- `ControlledUrlInput` replaces the bare `URLInput` usage in both `LinkControl` and `ToolbarLink`; it handles URL normalization internally (internal → relative, external → https://…), so callers no longer need to call `normalizeExternalUrl` / `toRelativeURL` themselves.
- `urlIsExternal(href)` (simple `http`-prefix check) used in `LinkControl` / `ToolbarLink` for auto-setting `target="_blank"` — documented comment warns it's used only for mega-menu-style links where internal URLs are already stripped of http(s).
- `LinkControl` moved from `src/components/LinkControl.tsx` to `src/components/editor-sidebar/LinkControl.tsx`.
- `setLinkTarget(link, undefined)` clears `target` and sets `rel` to `undefined` (not `noopener noreferrer`) — intended behaviour for removing external-link flags.
- `DragAndDropLink` (= `ILink & { id: string }`) defined in `frontend/lib/interfaces/link.ts` — id used for dnd-kit key tracking inside `SortableLinksModal`.
- `frontend/lib/type-guards/link.ts` added with `LinkTarget` type and `isLinkTarget` guard.
