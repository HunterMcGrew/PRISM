# Branch Plan: robertsamuel/thr-1419-custom-phone-numbers-in-click-to-call

## Goal

Allow editors to configure click-to-call CTAs to use either location phone numbers or a custom phone number list (stored in Thrive Platform Settings), and optionally pin a single specific number for direct dial instead of showing a modal picker.

---

## Current Status

Implementation complete. PR #1618 is open against `main`. Code review requested changes.

---

## Constraints / Intentional Decisions

- `custom:<index>` key scheme is intentional — mirrors PHP ACF repeater index. Known limitation: reordering the custom list invalidates saved keys.
- The `ButtonGroupItemBlock` fallback that links to `firstPhone` when no `selectedPhoneKey` is set is carry-over from the old behaviour (always linking to `locations[0].phone`). Intentional but inconsistently documented.
- `ISiteSettings.customPhoneNumbers` is optional (`?`) because it may not exist in older cached/serialized settings; `AppContextPageProps` requires it because it is always defaulted before reaching context.

---

## Do Not Undo

- Do not remove the `?? []` fallback in `getAppContextProps` — `ISiteSettings.customPhoneNumbers` is optional.
- Do not remove the early-return guard in `ButtonGroupItemBlock.handleClick` for `getCallDirectHref` — it prevents `e.preventDefault()` from blocking native `tel:` link navigation.
- Do not collapse `OpenModalArgs` union back to a single type — the discriminated union enables type-safe `phoneSource` on the `call` variant.

## Changes Made

- Added ACF field group `group_custom_phone_numbers` for platform settings repeater (name + phone, max 50).
- Added `cta_phone_source` (select: locations/custom) and `cta_selected_phone_key` (dynamic select) ACF fields to primary and secondary CTA repeaters.
- Registered GraphQL `CustomPhoneNumber` type and `customPhoneNumbers` field on `ThriveSettings`.
- Added `populateSelectedPhoneKeyChoices()` to `CallToActions` PHP class to fill the admin dropdown.
- Created `phone-source-utils.ts` with `getPhoneListFromSource`, `resolvePhoneFromKey`, `phoneToTelHref`, `getCallDirectHref`.
- Created `CallPhoneSelectControls` shared editor component for phone source + number dropdowns.
- Threaded `phoneSource` and `selectedPhoneKey` through block attributes, header CTA types, `ICallToAction`, `ModalState`, and `OpenModalArgs`.
- Wired `customPhoneNumbers` from GraphQL → `ISiteGlobals` → `AppContextPageProps` → `AppContext`.
- Updated `SharedModals` to render custom phone list when `modal.phoneSource === "custom"`.
- Updated `useCTAAction` hook: `isNavigationCTA` now returns true for call CTAs with a direct href; added `getCTAHref` for use in header link components.
- Updated `IconButton` to accept `href` prop and render as `<a>` when provided.
