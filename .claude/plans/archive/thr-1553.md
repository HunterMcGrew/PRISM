## Goal

Fix the Mega Menu Editor block sidebar inspector that gets cut off, preventing access to some controls.

---

## User Stories

Not applicable — this is a bug fix.

## Decisions

- Sidebar must be 320px (fixed) — this was the original WordPress `InterfaceSkeleton` width before the WP 6.9 private-apis removal forced a custom layout. The custom layout incorrectly set 280px in `styles.scss` and 300px in `sidebar/styles.scss`. Both must be 320px.
- Sidebar must never be larger than 320px — use `max-width: 320px` to enforce.
- Sidebar must support the collapse/expand transition — `aria-expanded` toggling with `transform: translateX` must continue to work.
- Vertical overflow fix: add `min-height: 0` to `&__sidebar` and `&__secondary-sidebar` in the `.interface-interface-skeleton` block so `overflow: auto` activates.
- Fix goes in `styles.scss` `&__sidebar` block (line 128), not in the group selector at line 215 — the group selector bundles `flex: 1` and `font-family` which the sidebar doesn't need.

## Debugged Issues

### Sidebar cut off — width mismatch + missing min-height: 0
- **Status:** `fixed`
- **Fixed in:** `_variables.scss` ($sidebar-width → 20rem/320px), `styles.scss` (&__sidebar width → $sidebar-width + max-width + min-height: 0, &__secondary-sidebar + min-height: 0)
- **Severity:** Medium
- **Environment:** WordPress admin — Mega Menu Editor custom app (`/wp-admin/admin.php?page=mega-menu-editor`)
- **Files:**
  - `backend/plugins/gravity-platform-core/src/apps/mega-menu-editor/styles.scss:128-134` — `__sidebar` width and flex constraints
  - `backend/plugins/gravity-platform-core/src/apps/mega-menu-editor/components/sidebar/styles.scss:59` — panel width
  - `backend/plugins/gravity-platform-core/src/apps/mega-menu-editor/_variables.scss:4` — `$sidebar-width` variable
- **Root cause:** Two issues:
  1. **Width mismatch:** The original WordPress `InterfaceSkeleton` provided a 320px sidebar. The custom replacement sets `__sidebar` to 280px (styles.scss:130) and the inner panel to 300px ($sidebar-width in _variables.scss:4). Both are wrong — the sidebar was 320px and should remain 320px.
  2. **Vertical overflow:** `__sidebar` lacks `min-height: 0`. As a flex child of `__body` (flex row, `overflow: hidden`), the sidebar's `min-height` defaults to content size, preventing `overflow: auto` from activating. The parent clips the overflow vertically.
- **Steps to Reproduce:**
  1. Navigate to the Mega Menu Editor admin page
  2. Select a block with many sidebar controls (e.g. Mega Menu Editor block — 4 color pickers + descriptions)
  3. Open the Settings sidebar (gear icon)
  4. Observe the sidebar content is cut off — right edge clipped (280px container vs 300px panel) and bottom controls unreachable (no vertical scroll)
- **Expected behavior:** The sidebar is 320px wide and scrolls vertically to reveal all inspector controls
- **Actual behavior:** The sidebar is 280px with a 300px panel inside (horizontal clipping). Vertical content below the fold is clipped with no scrollbar.
- **Recommended fix:**
  1. Set `$sidebar-width` to `320px` (20rem) in `_variables.scss`
  2. Set `&__sidebar` width to `$sidebar-width` in `styles.scss` (replace hardcoded 280px)
  3. Add `min-height: 0` to `&__sidebar` and `&__secondary-sidebar` for vertical scrolling
  4. Verify the `aria-expanded` transition still collapses/expands correctly
- **Suggested tests:** Open sidebar with a block that has extensive InspectorControls. Verify it's 320px wide, scrolls vertically, and collapses/expands via the gear toggle.
- **Linear:** `synced`
