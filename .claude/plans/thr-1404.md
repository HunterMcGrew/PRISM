# Branch Plan: robertsamuel/thr-1404-offers-carousel-arrows-overlapping-cards

## Goal

Replace overlapping arrow navigation in OffersCarousel with a segments-based pagination variant on the shared Slider component.

---

## Decisions

- Segments pagination is opt-in via `paginationVariant="segments"` — only OffersCarousel uses it; all other sliders keep dots as the default.
- Arrows are always hidden when using segments variant — enforced both at call site and internally in Slider.
- `effectivePerPage` tracks Splide's responsive breakpoint via `onMounted`/`onUpdated` events to compute correct segment count at all viewport widths.
- `requestedSegmentPageRef` prevents stale state when Splide clamps the last page index.
- `mb-6` on OffersCarousel wrapper is intentional per design spec (was `mb-3`).
- `<nav>` replaced with `<div role="group">` per review — segments are not a primary navigation landmark.
