# Branch Plan: hmcgrew/thr-1400-replace-urlinputs-with-controlledurlinput-component-in

## Goal

Adopt `ControlledUrlInput`, `LinkControl`, and `IconSelectionGrid` in existing Gutenberg editor blocks while keeping compatibility with both legacy string links and `ILink` object links.

---

## Decisions

- Keep `LinkValue` support (`string | ILink`) in editor controls, and normalize to `ILink` only at attribute write boundaries where object storage is expected.
- Use `ControlledUrlInput` as the URL entry surface for sidebar and toolbar link editing to centralize URL normalization behavior.
- Reuse shared sidebar controls (`LinkControl`, `IconSelectionGrid`, `ColorSelectionButton`) instead of block-local one-off UI.

---

- 2026-03-12: Branch reviewed after merging dependency PRs `#1615` and `#1614`.
- 2026-03-12: Fixed sortable-links label authority by disabling `LinkControl` label editing in `EditLinkScreen` and keeping `Title` as the single source of truth.

PR description rewritten to reflect full scope and precise merge order.
