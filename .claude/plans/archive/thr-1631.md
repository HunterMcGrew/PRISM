## Decisions

- **Clove creates the PR without prompting.** The "Want me to push this up?" prompt fails the Round 10 three-question test (reversible + predictable + in-lane → all yes → action, not prompt). The two-path closing after PR creation is preserved because it's a genuine decision point for the user (review now or later).
- **Briar does not absorb PR creation.** When the user invokes Briar without Clove/Eli having shipped first, Briar routes back to the author persona rather than stretching her role. This preserves the author-vs-reviewer separation that's load-bearing across the ecosystem.
- **"Authors ship, reviewers review."** The principle is documented in AGENTS.md § 0 so skill authors have one place to see the rule. It applies uniformly: Clove ships code PRs, Eli ships docs PRs, Sage and Reese ship artifact PRs. Briar and Eric don't ship — they review.
- **Release PR ownership is unchanged.** Team lead still owns release PRs. Sage and Reese produce artifact PRs that the release PR consumes; they don't produce release PRs.
- **THR-1630 coordination, not blocking.** Reese's input-handling expansion (PR, PR group, tag range) is separate work. This ticket lands her shipping flow at current scope; THR-1630 expands her input surface later without re-doing shipping.
- **Round 10 addition ships with this ticket.** The framework lives in the audit strategy plan; this ticket implements it. Bundling them in one PR gives the reviewer full context — the "why" and the "what" arrive together.
