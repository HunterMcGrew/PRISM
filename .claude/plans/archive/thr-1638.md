## Decisions

- **Keep `## Summary` as a one-line TL;DR above What/Why/How.** Reason: Hunter's call — a TL;DR line gives scanners immediate orientation without forcing them into the three-section narrative. One sentence avoids the overlap problem because it's explicitly a condensed pointer, not a duplicate.
- **Drop the current "2–3 sentence Summary" framing.** Reason: that framing collides with What/Why. Summary is now explicitly one line — the three-section narrative carries the detail.
- **Drop `## Changes`.** Reason: the GitHub diff (Files changed tab) already shows file count, tree, per-file line counts, and content — a prose list duplicates it and drifts when late commits land. All three research sources (HackerOne, Juro, Blockly) explicitly advise against per-file inventories in the PR body. Juro calls out "Changed a variable name / Updated some CSS" as anti-examples. Narrative covers decisions the diff can't show; the diff covers the rest.
- **Drop `## Acceptance Criteria` from the PR body.** Reason: AC is already maintained in the Linear ticket and the branch plan per ADR-0009. Duplicating it into the PR body creates drift between three sources — Linear, plan, and PR. The PR's `## Ticket` link points a reviewer to the authoritative AC.
- **Add `## Screenshots` as a named section** (deletable when not applicable). Reason: today it's ad-hoc; the rule file already treats it as load-bearing by saying "preserve existing screenshot sections." Making it a named, deletable section closes that gap.
- **Use `## Notes`, not `## Anything Else?`.** Reason: statement heading invites content; question heading nudges toward "no."
- **Intentionally diverge from `.github/pull_request_template.md`.** Reason: the sparse GitHub template becomes a signal. A PR body using the richer agent-generated structure means the skill ran; a PR body matching the sparse template means it was opened cold. Leaving the GitHub template untouched preserves that tell. This divergence must be documented in `.claude/rules/pr-description.md` so a future agent doesn't "fix" the drift by syncing the files.
- **Ship template + rule file together.** Reason: the rule file currently instructs agents to "collapse into Summary + Changes" — shipping the template without updating the rule creates silent contradiction that confuses Clove, Briar, and Eric.
- **Express optionality via placeholder text, not structural branching.** Reason: templates stay linear and scannable; "[Skip if trivial]" placeholders teach contributors what's expected and signal that empty sections are acceptable when honest.
- **Expand scope to drop the `"file-by-file Changes section"` clause from the DX ticket-types entry.** Reason: same drift class as the PR-body `## Changes` section this PR kills — Briar's review flagged it, Hunter pulled it into scope. Narrow fix only: remove the stale clause, keep the "Currently X / Makes it Y / because Z" spine. The broader "apply What/Why/How to ticket templates across all four types" work remains out of scope for a future ticket.
