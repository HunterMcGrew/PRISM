# Plan: THR-1530

## Ticket

https://linear.app/tractru/issue/THR-1530/document-hosting-infrastructure-and-multi-tenant-architecture

## Goal

Create a single infrastructure reference page so new developers can understand the hosting model, per-dealer isolation, and local development setup without piecing together stale docs.

---

## Decisions

- Infrastructure doc lives at `docs/content/dev/architecture/infrastructure.md` — architecture is the right category for hosting model documentation
- Stale ops docs (`go-live-procedure.md`, `dealer-site-spinup.md`) keep their WARNING banners but get an added cross-reference to the new infrastructure page — they still have value as legacy reference
- The `/thrive` redirect is implemented via Next.js middleware using `BACKEND_PATHS` constant, not a Next.js redirect config — document accurately
- Domain conventions include both `thrivewebsiteadmin.com` (Kubota/Thrive) and `sieqwebsiteadmin.com` (AGCO/SIEQ) — document both
- No `.claude/architect/` context file needed for this doc — it describes infrastructure, not code patterns
